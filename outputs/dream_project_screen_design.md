# 29 WITH - 꿈프 화면 설계

## 1. 화면 설계 원칙

이번 화면 설계의 기준은 “많이 보여주는 것”이 아니라 “바로 판단하고 바로 행동하게 하는 것”이다.

공통 원칙:

- 한 화면의 핵심 행동은 1개만 둔다.
- 보조 행동은 접거나 우측/하단에 작게 둔다.
- 설명 문구는 최소화하고, 상태와 숫자로 이해하게 한다.
- 관리자가 판단해야 하는 정보와 선생님이 안심해야 하는 정보를 섞지 않는다.
- 완료된 항목은 작게, 문제 있는 항목은 크게 보여준다.
- 버튼은 현재 상태에서 누를 수 있는 것만 보여준다.

버튼 원칙:

- 주요 버튼: 지금 가장 해야 하는 행동
- 보조 버튼: 보기, 다운로드, 문의
- 위험 버튼: 종료, 취소, 삭제

피해야 할 버튼:

- `관리`
- `처리`
- `확인`
- `진행`

권장 버튼:

- `신청서 제출`
- `선정 처리`
- `출품 확인 시작`
- `작품 상태 보기`
- `활동확인서 PDF 생성`
- `간식비 지급 예정일 입력`
- `관리자에게 확인 요청`

---

## 2. 전체 구조

29 WITH 진입 후 관리자와 선생님은 서로 다른 화면 구조를 가진다.

관리자:

```text
전체 대시보드
꿈프
  - 꿈프 대시보드
  - 행사 운영
  - 신청/선정
  - 출품 확인
  - 혜택/지원
  - 활동확인서
  - 심사 결과
  - 문의/요청
  - 히스토리
```

선생님:

```text
내 대시보드
신청 가능한 행사
참여 프로젝트
학생/작품 관리
혜택/지원
공지/메일함
활동확인서
심사표
참여 이력
내 프로필
```

관리자 화면은 “처리 중심”, 선생님 화면은 “확인 중심”이다.

## 3. 관리자 화면

### 3.1 관리자 - 전체 대시보드

목적:

- 29 WITH 전체 운영 상태 확인
- 오늘 처리할 일이 있는 카테고리로 이동

상단:

```text
29 WITH
전체 | 꿈프 | 서포터즈 | 명예홍보대사 | 파트너사
```

본문 구성:

```text
[오늘 처리 필요]
꿈프 9건
서포터즈 0건
명예홍보대사 0건
파트너사 0건

[운영 중인 프로젝트]
꿈프 3개 행사 운영 중
서포터즈 설계 예정
명예홍보대사 설계 예정
파트너사 설계 예정
```

주요 버튼:

- `꿈프 보기`

과감히 제거:

- 긴 서비스 설명
- 큰 카테고리 버튼
- 내부 정책 안내
- 선택 방법 설명

UX:

- 전체 대시보드는 요약만 보여준다.
- `꿈프`를 누르면 바로 꿈프 대시보드로 들어간다.

---

### 3.2 관리자 - 꿈프 대시보드

목적:

- 현재 진행 중인 꿈프 행사별 상태를 한눈에 확인
- 먼저 처리해야 할 행사 선택

화면 구성:

```text
꿈프 대시보드

[진행 중인 꿈프]
제13회 박카스 29초영화제
선정 28교 · 출품 확인 104/122편 · 처리 필요 6건
[출품 확인]

제1회 동반성장 29역숏폼왕
선정 14교 · 출품 확인 47/64편 · 처리 필요 3건
[출품 확인]

간단요리사 29역숏폼왕
선정 9교 · 출품 확인 31/36편 · 처리 필요 없음
[상세 보기]
```

하단:

```text
[오늘 할 일]
- 미확인 작품 9건
- 메일 재발송 필요 2건
- 간식비 지급 예정일 미입력 4교
```

주요 버튼:

- `출품 확인`
- `상세 보기`

과감히 제거:

- 전체 통계 과다 노출
- 다른 프로젝트 카테고리 상세 정보
- 긴 연동 안내 문구

UX:

- 행사는 카드형으로 보여준다.
- 처리 필요 건이 있는 행사를 먼저 정렬한다.
- 행사 카드를 누르면 해당 행사 기준 화면으로 이동한다.

---

### 3.3 관리자 - 행사 운영

목적:

- 선택한 행사 하나의 운영 단계 확인
- 다음 해야 할 일을 처리

화면 구성:

```text
제13회 박카스 29초영화제
모집중 · 외부 연동 정상 · 출품 마감 D-5

[운영 단계]
1. 행사 등록 완료
2. 꿈프 모집 시작 완료
3. 신청/선정 완료
4. 출품 자동 확인 진행 중
5. 최종 리스트 승인 대기
6. 활동확인서 발급 전
7. 심사표 반영 전
8. 종료 전
```

상단 주요 버튼:

- 현재 단계에 맞는 버튼 1개만 노출

예시:

- 모집 전: `꿈프 모집 시작`
- 신청 기간: `신청자 보기`
- 선정 전: `선정 처리`
- 출품 중: `출품 확인`
- 마감 후: `최종 리스트 승인`
- 승인 후: `활동확인서 PDF 생성`
- 심사 후: `심사표 생성`
- 모든 완료 후: `꿈프 종료`

보조 버튼:

- `행사 정보 수정`
- `메일 기록 보기`
- `연동 설정`

과감히 제거:

- 한 화면에 모든 작업 버튼 노출
- 완료된 단계의 상세 설명
- 외부 서버 정보 직접 노출

UX:

- 관리자는 현재 단계에서 다음에 무엇을 해야 하는지만 보면 된다.
- 완료된 단계는 펼치기 전에는 상세가 보이지 않게 한다.

---

### 3.4 관리자 - 신청/선정

목적:

- 신청 학교를 빠르게 검토하고 선정 처리

화면 구성:

```text
신청/선정

[요약]
신청 42교 · 목표 30교 · 예상 작품 186편 · 목표 200편

[필터]
전체 | 검토 필요 | 성실 참여 | 확인 필요 이력 | 선정 | 예비 | 미선정

[학교 리스트]
학교명 | 담당교사 | 예상 작품 | 참여 상태 | 신청 상태 | 다음 행동
방학중학교 | 김하은 | 5편 | 성실 참여 | 신청 접수 | [선정]
도봉고등학교 | 이다온 | 8편 | 확인 필요 | 검토 필요 | [상세 보기]
```

학교 상세 패널:

```text
학교명
담당교사 / 연락처 / 이메일
출품 소속명
예상 출품편수
활용 계획
참여 이력
관리자 메모

[선정 처리] [예비 처리] [미선정 처리]
```

주요 버튼:

- `선정 처리`

보조 버튼:

- `예비 처리`
- `미선정 처리`
- `메일 재발송`
- `엑셀 다운로드`

과감히 제거:

- 리스트에 긴 활용 계획 전체 노출
- 통장사본/신분증 원본 바로 노출
- 쿠폰번호를 선정 전 리스트에 노출

UX:

- 리스트는 검토용, 상세 패널은 판단용이다.
- 복수 선택은 가능하되, 선정 처리 전 확인 팝업을 띄운다.
- 선정 처리 결과를 팝업에서 미리 보여준다.

---

### 3.5 관리자 - 출품 확인

목적:

- 학교별 출품 확인 상태를 빠르게 보고 미확인 건 조치

화면 구성:

```text
출품 확인

[요약]
선정 28교 · 신청 122편 · 자동 확인 104편 · 미확인 18편

[필터]
전체 | 미확인 있음 | 자동 확인 완료 | 수동 확인 필요 | 선생님 확인 요청 중

[학교 리스트]
학교명 | 신청 | 확인 | 미확인 | 상태 | 다음 행동
방학중학교 | 5 | 4 | 1 | 확인 필요 | [작품 보기]
서울영상고 | 8 | 8 | 0 | 완료 | [보기]
```

학교 상세:

```text
방학중학교
출품 소속명: 방학중학교

작품 1 확인 완료
작품 2 확인 완료
작품 3 확인 완료
작품 4 확인 완료
작품 5 미확인

[선생님에게 확인 요청] [작품 링크 추가]
```

주요 버튼:

- `작품 보기`
- `선생님에게 확인 요청`

보조 버튼:

- `전체 동기화`
- `선택 학교 재확인`
- `수동 매칭`

과감히 제거:

- 첫 화면에서 작품 전체 리스트 노출
- 모든 출품작 썸네일 노출
- 연동 로그 상시 노출

UX:

- 학교당 작품이 많으므로 첫 화면은 반드시 학교 단위로 보여준다.
- 작품 상세는 학교를 클릭했을 때만 보여준다.
- 미확인 작품이 없는 학교는 작게 접는다.

---

### 3.6 관리자 - 혜택/지원

목적:

- 구독권과 간식비 상태를 학교별로 관리

화면 구성:

```text
혜택/지원

[요약]
구독권 지급 완료 28교
간식비 지급 예정일 입력 21교
미입력 7교

[학교 리스트]
학교명 | 구독권 | 간식비 상태 | 예정일 | 금액 | 다음 행동
방학중학교 | 지급 완료 | 지급 예정 | 2026.07.15 | 150,000원 | [수정]
서울영상고 | 지급 완료 | 미입력 | - | - | [예정일 입력]
```

주요 버튼:

- `간식비 지급 예정일 입력`
- `지급 완료 처리`

보조 버튼:

- `쿠폰 업로드`
- `쿠폰 재고 보기`
- `지원 안내 메일 발송`

과감히 제거:

- 선생님 화면에 필요 없는 정산 내부 정보
- 회계 처리 상세 로그

UX:

- 선생님에게 보일 문구를 관리자 화면에서 미리 확인할 수 있게 한다.
- 간식비 미입력 학교만 빠르게 필터링할 수 있어야 한다.

---

### 3.7 관리자 - 활동확인서

목적:

- 출품 마감 후 최종 리스트 승인
- 활동확인서 PDF 생성 및 발송

화면 구성:

```text
활동확인서

[요약]
승인 대기 8교 · 발급 완료 20교

[필터]
승인 대기 | PDF 생성 필요 | 발급 완료 | 재발급 요청

[학교 리스트]
학교명 | 최종 작품 수 | 학생 이름 확인 | 승인 상태 | 확인서
방학중학교 | 5편 | 완료 | 승인 대기 | [미리보기]
```

상세:

```text
방학중학교
작품명 5개
학생 이름 7명
활동 기간

[최종 승인] [PDF 미리보기]
```

주요 버튼:

- `최종 승인`
- `활동확인서 PDF 생성`

보조 버튼:

- `PDF 미리보기`
- `발급 메일 발송`
- `재발급`

과감히 제거:

- 승인 전 다운로드 버튼
- 템플릿 설정 버튼 상시 노출

UX:

- 승인 전에는 미리보기만 가능하다.
- 승인 후에만 PDF 생성과 발송 버튼이 보인다.

---

### 3.8 관리자 - 심사 결과

목적:

- 수상작 발표 후 순위와 본심 여부 반영

화면 구성:

```text
심사 결과

[요약]
결과 반영 완료 18교 · 반영 필요 10교

[학교 리스트]
학교명 | 작품 수 | 순위 반영 | 본심 반영 | 심사표
방학중학교 | 5편 | 완료 | 완료 | [PDF 생성]
```

주요 버튼:

- `심사 결과 동기화`
- `심사표 PDF 생성`

보조 버튼:

- `PDF 미리보기`
- `결과 안내 메일 발송`

과감히 제거:

- 선생님에게 필요 없는 원본 점수 노출
- 심사 내부 값 노출

UX:

- 29역숏폼왕에서는 본심 열을 숨긴다.
- 결과 반영 전에는 PDF 버튼을 비활성화한다.

---

### 3.9 관리자 - 문의/요청

목적:

- 선생님이 보낸 요청을 빠르게 처리

화면 구성:

```text
문의/요청

[필터]
전체 | 학교명 변경 | 작품 확인 | 확인서 재발급 | 간식비 문의 | 처리 완료

[요청 리스트]
요청일 | 학교명 | 유형 | 내용 요약 | 상태 | 다음 행동
2026.07.08 | 방학중학교 | 작품 확인 | 1편 미확인 문의 | 접수 | [답변]
```

주요 버튼:

- `답변 등록`
- `승인`
- `반려`
- `처리 완료`

과감히 제거:

- 관리자 내부 메모 자동 노출
- AI 채팅형 명령을 기본 조치로 제공

UX:

- 요청 유형을 정해두면 관리자가 빠르게 처리할 수 있다.
- 자유 입력은 보조로만 둔다.

---

### 3.10 관리자 - 히스토리

목적:

- 종료된 꿈프 보관 및 조회

화면 구성:

```text
히스토리

[종료된 꿈프]
행사명 | 기간 | 선정 학교 | 최종 출품 | 확인서 | 심사표 | 종료일
제12회 신한 29초영화제 | 2026.04~05 | 32교 | 156편 | 32건 | 32건 | 2026.07.01
```

주요 버튼:

- `상세 보기`
- `운영 기록 다운로드`

보조 버튼:

- `활동확인서 다운로드`
- `심사표 다운로드`

과감히 제거:

- 종료된 행사를 진행 중 메뉴에 계속 노출

UX:

- 종료된 꿈프는 운영 화면에서 빠지고 히스토리에만 남는다.
- 선생님은 본인 문서를 계속 다운로드할 수 있어야 한다.

## 4. 선생님 화면

### 4.1 선생님 - 내 대시보드

목적:

- 신청이 잘 들어갔는지 확인
- 현재 선정/출품/혜택 상태 확인
- 내가 해야 할 일 확인

화면 구성:

```text
내 대시보드

제13회 박카스 29초영화제
신청 접수 완료 · 선정 완료 · 출품 마감 D-5

[현재 상태]
신청 5편
출품 확인 4편
미확인 1편

[해야 할 일]
미확인 작품 1편의 출품 상태를 확인해 주세요.

[혜택/문서]
구독권 지급 완료
간식비 지급 예정: 2026.07.15
활동확인서: 관리자 확인 중
심사표: 발표 후 제공
```

주요 버튼:

- `작품 상태 보기`

보조 버튼:

- `신청서 보기`
- `소속명/팀명 안내`
- `문의하기`

과감히 제거:

- 전체 행사 통계
- 다른 학교 정보
- 관리자 내부 상태
- 긴 정책 설명

UX:

- 문제가 없으면 “정상 진행 중”을 가장 크게 보여준다.
- 문제가 있으면 할 일을 먼저 보여준다.
- 선생님 첫 화면에서는 버튼을 많이 두지 않는다.

---

### 4.2 선생님 - 신청 가능한 행사

목적:

- 신청 가능한 꿈프 확인
- 행사 상세 확인 후 신청

화면 구성:

```text
신청 가능한 행사

제13회 박카스 29초영화제
29초영화제 · 모집중 · 공모기간 2026.04.08~2026.05.21
[자세히 보기] [신청하기]

간단요리사 29역숏폼왕
29역숏폼왕 · 모집 예정
[자세히 보기] [신청 대기]
```

주요 버튼:

- `신청하기`

보조 버튼:

- `자세히 보기`

비활성 버튼:

- `신청 대기`
- `신청 완료`

과감히 제거:

- 리스트에서 포스터 크게 노출
- 안내사항 전문 노출
- 홈페이지 URL 전체 텍스트 노출

UX:

- 리스트에는 행사명, 유형, 기간, 상태만 보여준다.
- 자세한 내용은 팝업 또는 상세 화면에서 보여준다.
- 모집 전에는 신청 버튼을 비활성화한다.

---

### 4.3 선생님 - 행사 상세/신청

목적:

- 신청 전 핵심 정보 확인
- 출품 소속명/팀명 중요 안내
- 신청서 제출

상세 화면 구성:

```text

행사명
행사 유형
공모기간
주제
안내사항

[홈페이지 보기] [신청하기]
```

신청 화면 구성:

```text
프로필

학교급
담당교사명
연락처
이메일

[방학중학교]

신청 정보
예상 출품편수
활용 계획
출품 예정일
메모

[신청서 제출]
```

중요 안내:

```text
출품 소속명/팀명은 실제 출품 시 입력하는 이름과 반드시 동일해야 합니다.

```

주요 버튼:

보조 버튼:

- `임시저장`
- `이전`

과감히 제거:

- 회원가입 때 학교 정보 입력
- 신청서에 불필요한 긴 설명 반복
- 관리자 내부 참여 이력 상세

UX:

- 출품 소속명/팀명 입력란은 가장 눈에 띄게 배치한다.
- 입력한 이름을 복사할 수 있게 한다.
- 제출 후에는 신청 완료 화면을 반드시 보여준다.

## 6. 최종 정리

이 화면 설계에서 가장 중요한 판단은 아래와 같다.

관리자:

- 전체 작품 리스트보다 학교 단위 요약을 먼저 보여준다.
- 다음에 해야 할 버튼 1개를 가장 크게 둔다.
- 상세 정보는 선택했을 때만 보여준다.

선생님:

- 신청 접수, 선정 상태, 출품 확인, 혜택 상태를 첫 화면에서 바로 보여준다.
- 불필요한 내부 정보는 숨긴다.
- 문제가 있으면 설명보다 조치 버튼을 먼저 보여준다.
