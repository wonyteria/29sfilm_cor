# PRD - 29 Platform
## Dream Project(꿈나무양성프로젝트) - 관리자 작업함 및 예외 조치 프로세스

---

## 1. 프로젝트 개요

본 문서는 꿈나무양성프로젝트 운영 중 발생하는 실패, 누락, 충돌, 연동 오류를 관리자가 한눈에 확인하고 빠르게 조치할 수 있는 관리자 작업함 기능을 정의한다.

핵심 목적:

- 운영 중 발생한 문제를 한 화면에 모아 표시
- 문제 유형별 권장 조치 제공
- 관리자가 빠르게 수동 보정할 수 있는 액션 제공
- 반복적 작업은 AI 보조 기능으로 처리 속도 향상
- 최종 변경은 관리자가 확인 후 실행

---

## 2. 기본 방향

관리자 조치 방식은 아래처럼 역할을 나눈다.

```text
수동 조치 = 최종 확정 및 데이터 변경
AI 보조 = 원인 분석, 후보 추천, 일괄 작업 초안 생성
```

정책:

- AI가 직접 최종 승인, 매칭 확정, 메일 발송, PDF 발급을 실행하지 않는다.
- AI는 후보 추천과 작업 초안을 제공한다.
- 관리자가 확인 버튼을 눌러야 실제 데이터가 변경된다.
- 모든 조치 이력을 저장한다.

---

- 권장 조치

문제 유형:

- 인증실패
- 연동실패
- 페이지구조변경

필터:

## 4. 문제 유형별 빠른 조치

### 4.1 연동정보미등록

권장 조치:

- 외부 관리자 서버 선택
- 출품작 관리 페이지 URL 입력
- 행사 식별값 매핑
- 연동 테스트 실행

빠른 액션:

- `연동 정보 등록`
- `연동 테스트`
- `자동 연동 제외`

### 4.2 인증실패

권장 조치:

- 로그인 정보 재입력
- 계정 잠금 여부 확인
- 비밀번호 변경 여부 확인

빠른 액션:

- `로그인 정보 수정`
- `재로그인 테스트`
- `관리자에게 알림`

### 4.3 페이지구조변경 또는 필수컬럼누락

권장 조치:

- 관리자 페이지 구조 확인
- 필수 컬럼 매핑 재설정
- 엑셀 다운로드 방식으로 전환

빠른 액션:

- `컬럼 매핑 수정`
- `엑셀 업로드로 대체`
- `연동 일시 중지`

### 4.4 매칭실패

권장 조치:

- 꿈프 학교명과 외부 소속명/팀명 비교
- 후보 출품작 검색
- 수동 매칭

빠른 액션:

- `후보 검색`
- `수동 매칭`
- `담당자에게 정정 요청 메일`
- `매칭 제외`

### 4.5 매칭충돌

권장 조치:

- 동일 학교명 또는 동일 소속명/팀명 신청 건 비교
- 올바른 신청 건 선택

빠른 액션:

- `후보 비교`
- `신청 건 선택 후 매칭`
- `중복 신청 표시`

### 4.6 이름정보누락

권장 조치:

- 29초영화제 감독명 확인
- 29역숏폼왕 출품자명 확인
- 필요 시 관리자 수동 입력

빠른 액션:

- `이름 직접 입력`
- `외부 페이지 다시 조회`
- `담당자에게 확인 요청 메일`

### 4.7 활동확인서 생성실패

권장 조치:

- 템플릿 설정 확인
- 누락 필드 확인
- PDF 재생성

빠른 액션:

- `템플릿 설정으로 이동`
- `누락 필드 확인`
- `PDF 재생성`

### 4.8 심사 결과 매칭실패

권장 조치:

- 작품 고유 ID 확인
- 작품 URL 확인
- 작품명/감독명 보조 비교
- 수동 매칭

빠른 액션:

- `후보 심사 결과 검색`
- `수동 매칭`
- `심사 결과 재동기화`

### 4.9 메일 발송실패

권장 조치:

- 이메일 주소 확인
- 메일 서버 상태 확인
- 재발송

빠른 액션:

- `이메일 수정`
- `메일 재발송`
- `담당자 연락처 확인`

## 5. AI 보조 기능

AI 보조는 채팅형 인터페이스로 제공할 수 있다.

가능한 역할:

- 실패 원인 요약
- 후보 출품작 추천
- 후보 매칭 비교표 생성
- 학교명과 소속명/팀명 차이 설명
- 메일 문안 초안 작성
- 일괄 처리 대상 추천
- 수동 입력값 검토

AI 명령 예시:

```text
이 매칭 실패 건의 후보를 찾아줘.
```

```text
서울고등학교와 서울 고등학교 차이 때문에 실패한 건만 모아줘.
```

```text
메일 발송 실패 건 중 이메일 형식이 이상한 항목을 찾아줘.
```

```text
이 담당자에게 보낼 정정 요청 메일 초안을 만들어줘.
```

제한:

- AI는 최종 매칭을 확정하지 않는다.
- AI는 승인 버튼을 대신 누르지 않는다.
- AI는 메일을 자동 발송하지 않는다.
- AI 제안은 관리자가 검토 후 적용한다.

권장 UX:

- 각 예외 상세 화면에 `AI에게 원인 분석 요청` 버튼 제공
- AI 답변 아래에 `제안 적용`, `수동 수정`, `무시` 버튼 제공
- `제안 적용`을 누르면 변경 전 확인 모달을 표시

---

## 6. 일괄 조치

관리자는 동일 유형의 문제를 복수 선택하여 일괄 조치할 수 있다.

가능한 일괄 조치:

- 메일 재발송
- 외부 데이터 재동기화
- PDF 재생성
- 담당자 정정 요청 메일 발송
- 처리 상태 변경
- 무시 처리

정책:

- 데이터 변경이 있는 일괄 조치는 실행 전 확인 모달을 표시한다.
- 일괄 조치 결과를 성공/실패로 나누어 기록한다.
- 실패 건은 다시 작업함에 남긴다.

## 7. 이력 관리

저장 항목:

- 문제 발생 시각
- 문제 유형
- 원본 오류 메시지
- 관리자 조치 내용
- AI 제안 내용
- 적용 여부
- 처리자
- 처리일시
- 처리 결과

정책:

- AI 제안을 적용한 경우 제안 원문과 적용 결과를 저장한다.
- 수동 입력값은 변경 전/후 값을 저장한다.
- 관리자 작업함의 완료 처리는 이력 삭제가 아니라 상태 변경으로 처리한다.
