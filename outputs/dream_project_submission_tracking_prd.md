# PRD - 29 Platform
## Dream Project(꿈나무양성프로젝트) - 출품 진행 관리 및 자동 매칭 프로세스

---

## 1. 프로젝트 개요

본 문서는 꿈나무양성프로젝트 선정 이후, 담당 선생님이 실제로 29초영화제 또는 29역숏폼왕에 작품을 출품하는 과정을 관리하기 위한 기능을 정의한다.

핵심 목적은 아래와 같다.

- 출품 마감 전 담당 선생님에게 자동 안내 메일 발송
- 행사 홈페이지 서버와 연동하여 실제 출품 작품 자동 조회
- 꿈프 학교명과 외부 출품 데이터의 소속명 또는 팀명 기준으로 작품 자동 매칭
- 매칭된 출품 URL을 담당자 페이지와 관리자 화면에 표시
- 담당 선생님과 관리자가 출품 진행 상황을 쉽게 확인

본 단계는 접수 및 선발 프로세스 이후부터 출품 마감까지의 운영을 다룬다.

---

## 2. 사용자 유형

### 2.1 관리자

29초영화제 사무국 담당자.

권한:

- 출품 진행 현황 조회
- 자동 매칭 결과 조회
- 수동 매칭 및 매칭 해제
- 출품 URL 확인
- 마감 안내 메일 발송 이력 확인
- 메일 재발송
- 외부 행사 서버 연동 정보 등록 및 관리

### 2.2 꿈나무양성프로젝트 담당자

선정된 학교 선생님 또는 기관 담당자.

권한:

- 선정된 프로젝트 조회
- 출품 마감일 확인
- 자동 매칭된 작품 목록 확인
- 출품 URL 확인
- 직접 입력한 작품 링크와 자동 매칭 결과 비교
- 미매칭 상태 확인

---

## 3. 적용 대상

본 기능은 아래 조건을 만족하는 신청 건에 적용한다.

- 꿈나무양성프로젝트 신청 상태가 `선정`
- 담당자 페이지가 생성된 상태
- 연결된 행사의 상태가 `모집중` 또는 `출품 진행중`
- 행사에 출품 URL 또는 외부 서버 연동 정보가 등록된 상태

---

## 4. 출품 마감 안내 메일

출품 마감일이 다가오면 선정된 꿈나무양성프로젝트 담당 선생님에게 자동 안내 메일을 발송한다.

발송 기준:

- D-14
- D-10
- D-5
- D-1

기준일:

- 행사 관리에 등록된 공모기간의 종료일 또는 별도 등록된 출품 마감일을 기준으로 한다.
- 별도 출품 마감일이 등록되어 있으면 출품 마감일을 우선 사용한다.
- 별도 출품 마감일이 없으면 공모기간 종료일을 사용한다.

발송 대상:

- 해당 행사 꿈프 신청 상태가 `선정`인 담당자
- 선정 취소, 미선정, 취소 상태는 제외

메일 포함 내용:

- 행사명
- 출품 마감일
- 남은 기간
- 신청 작품 수
- 현재 자동 매칭된 작품 수
- 출품 URL
- 꿈프 학교명
- 29초영화제 소속명 또는 29역숏폼왕 팀명 동일 입력 안내
- 담당자 페이지 바로가기

메일 예시:

```text
안녕하세요. 29초영화제사무국입니다.

[행사명] 출품 마감이 D-{남은일수}일 남았습니다.

꿈나무양성프로젝트 신청 시 입력하신 학교명은 아래와 같습니다.

학교명: {학교명}

작품 출품 시 29초영화제의 소속명 또는 29역숏폼왕의 팀명에 해당 학교명을 반드시 동일하게 입력해 주세요.
동일하지 않을 경우 출품 확인이 지연되거나 지원 대상에서 제외될 수 있습니다.

현재 시스템에서 확인된 출품 작품 수: {매칭작품수}편
신청 작품 수: {신청작품수}편

담당자 페이지에서 출품 진행 상황을 확인해 주세요.
```

정책:

- 자동 메일은 발송 성공/실패 이력을 저장한다.
- 동일 신청 건에 같은 D-day 안내 메일은 1회만 자동 발송한다.
- 관리자는 특정 담당자에게 안내 메일을 재발송할 수 있다.
- 재발송은 자동 발송 이력과 구분하여 저장한다.
- 메일 발송 실패 시 관리자 화면에 실패 상태를 표시한다.

## 5. 외부 행사 서버 연동

29초영화제 또는 29역숏폼왕 홈페이지 서버에서 실제 출품 데이터를 조회하기 위한 연동 기능을 제공한다.

현재 서버 정보는 확정되지 않았으므로, 추후 서버 정보 입력만으로 작동할 수 있도록 연동 설정 구조를 먼저 마련한다.

연동 방식 우선순위:

1. 공식 API 연동
2. 관리자 페이지의 엑셀 다운로드 또는 데이터 다운로드 기능 연동
3. 관리자 페이지 검색 결과 기반 수집
4. 관리자가 엑셀 파일을 직접 업로드하는 수동 보완

권장 정책:

- 공식 API 또는 엑셀 다운로드 기능이 있으면 이를 우선 사용한다.
- 관리자 페이지 화면을 직접 읽어오는 방식은 화면 구조 변경에 취약하므로 보조 방식으로 사용한다.
- 화면 수집 방식으로 운영할 경우에도 반드시 행사 식별값과 검색 조건을 함께 사용해야 한다.
- 학교명만으로 전체 출품작을 검색해 자동 매칭하지 않는다.

### 5.1 연동 설정 항목

관리자는 행사별로 외부 서버 연동 정보를 등록할 수 있다.

연동 설정 항목:

- 연동 사용 여부
- 서버 유형
  - 29초영화제
  - 29역숏폼왕
  - 기타
- 29 Platform 행사 ID
- 외부 홈페이지 행사 ID
- 외부 홈페이지 행사명
- 서버 주소
- 인증 방식
  - API Key
  - Basic Auth
  - Bearer Token
  - 관리자 로그인 세션
  - VPN/내부망
  - 기타
- 인증 정보
- 출품 데이터 조회 경로
- 관리자 출품작 관리 페이지 URL
- 관리자 검색 파라미터
- 엑셀 다운로드 URL
- 작품 상세 URL 생성 규칙
- 소속명 필드명
- 팀명 필드명
- 작품명 필드명
- 출품자명 필드명
- 출품일 필드명
- 작품 URL 필드명
- 동기화 주기
- 마지막 동기화 시각

행사 유형별 필수 매칭 필드:

- 꿈프 기준값: 학교명
- 29초영화제 외부 데이터: 출품 소속명
- 29역숏폼왕 외부 데이터: 팀명

정책:

- 꿈프 신청 시 학교명은 필수로 입력한다.
- 자동 매칭은 꿈프 학교명과 29초영화제 소속명 또는 29역숏폼왕 팀명이 일치하는지 확인한다.
- 학교명 매칭은 띄어쓰기까지 포함한 완전 일치를 기본 기준으로 한다.
- 29역숏폼왕 관리자 페이지에 현재 팀명 컬럼이 없더라도, 추후 팀명 필드가 추가되면 해당 값을 필수 매칭 기준으로 사용한다.
- 팀명 필드가 추가되기 전까지는 자동 매칭을 확정 처리하지 않고 `수동확인필요` 또는 `팀명필드미등록` 상태로 표시한다.
- 출품자 회원명은 팀명을 대체할 수 없다.
- 작품 제목, 출품자명, 플랫폼 정보만으로는 꿈프 신청 건에 자동 매칭하지 않는다.

보안 정책:

- 인증 정보는 암호화하여 저장한다.
- 인증 정보는 관리자 화면에서도 원문을 다시 표시하지 않는다.
- 연동 테스트 기능을 제공한다.
- 연동 실패 이력을 저장한다.
- 관리자 로그인 방식 연동 시 계정 권한은 출품작 조회에 필요한 최소 권한으로 제한한다.

### 5.2 행사 식별값 매핑

29초영화제는 하나의 홈페이지에서 여러 영화제가 동시에 진행될 수 있으므로, 자동 매칭은 반드시 행사 식별값을 기준으로 분리한다.

필수 매핑:

- 29 Platform 행사 ID
- 외부 홈페이지 행사 ID 또는 관리자 페이지의 행사 식별값
- 외부 홈페이지 행사명
- 행사 유형
  - 29초영화제
  - 29역숏폼왕

예시:

```text
29 Platform 행사: A29초영화제
외부 홈페이지 행사 ID: festival_2026_a29
꿈프 학교명: 서울고
29초영화제 소속명: 서울고
```

```text
29 Platform 행사: B29초영화제
외부 홈페이지 행사 ID: festival_2026_b29
꿈프 학교명: 서울고
29초영화제 소속명: 서울고
```

위와 같이 동일 학교가 여러 행사에 참여하더라도 외부 홈페이지 행사 ID가 다르면 서로 다른 신청 건으로 분리하여 매칭한다.

정책:

- 외부 홈페이지 행사 ID가 없는 데이터는 자동 매칭하지 않는다.
- 행사 ID가 불명확한 경우 `수동확인필요` 상태로 분류한다.
- 학교명과 소속명/팀명이 같더라도 행사 ID가 다르면 서로 다른 출품으로 처리한다.
- 행사 ID와 학교명 매칭값이 모두 일치할 때만 자동 매칭한다.

### 5.3 관리자 페이지 기반 수집

공식 API가 없을 경우 관리자 페이지의 출품작 관리 화면에서 데이터를 가져오는 방식을 사용할 수 있다.

수집 방식:

- 행사별 관리자 출품작 관리 페이지 접속
- 행사 ID 또는 행사명으로 현재 행사 맥락 확인
- 검색 조건에 꿈프 학교명을 입력
- 검색 결과 목록에서 작품 정보 추출
- 필요 시 `보기` 버튼을 통해 작품 상세 URL 확인
- 필요 시 `엑셀 다운로드` 기능을 통해 전체 목록 수집

관리자 페이지에서 확인해야 하는 정보:

- 현재 접속한 행사명 또는 행사 ID
- 작품 번호
- 썸네일
- 등록일
- 출품자 또는 감독
- 29초영화제 출품 소속명
- 29역숏폼왕 팀명
- 분야
- 부문
- 작품제목
- 진행상태
- 수상결과
- 보기 URL
- 다운로드 URL

정책:

- 관리자 페이지 기반 수집 전 현재 페이지의 행사명이 29 Platform 행사와 매핑된 행사인지 확인한다.
- 행사명이 다르거나 확인할 수 없으면 수집하지 않는다.
- 검색 결과가 여러 건일 수 있으므로 작품 단위로 모두 저장한다.
- 화면 구조 변경으로 수집 실패 시 관리자에게 연동 실패로 표시한다.
- 가능하면 화면 목록보다 엑셀 다운로드 데이터를 우선 사용한다.
- 29역숏폼왕의 경우 팀명 컬럼 또는 팀명 검색 조건이 확인되지 않으면 자동 매칭하지 않는다.

### 5.4 연동 안정성 정책

외부 관리자 페이지 연동은 화면 구조, 로그인 세션, 권한, 네트워크 상태에 영향을 받으므로 안정성 정책을 별도로 둔다.

연동 방식 우선순위:

1. 공식 API
2. 엑셀 다운로드
3. 관리자 페이지 목록 수집
4. 관리자 수동 업로드

자동 재시도:

- 일시적 네트워크 오류는 최대 3회 재시도한다.
- 재시도 간격은 점진적으로 늘린다.
- 3회 실패 시 `연동실패`로 표시하고 관리자 작업함에 등록한다.

로그인/세션 정책:

- 세션 만료가 감지되면 1회 자동 재로그인을 시도한다.
- 재로그인 실패 시 `인증실패`로 표시한다.
- 비밀번호 변경 또는 계정 잠금으로 추정되는 경우 관리자에게 별도 알림을 표시한다.

페이지 구조 변경 감지:

- 필수 컬럼이 보이지 않으면 수집을 중단한다.
- 필수 버튼 또는 엑셀 다운로드 기능을 찾을 수 없으면 `페이지구조변경`으로 표시한다.
- 화면 구조가 바뀐 상태에서 임의로 데이터를 추정해 저장하지 않는다.

필수 컬럼:

- 29초영화제: 행사명 또는 행사 ID, 작품명, 감독, 소속명, 예심 평점, 수상결과
- 29역숏폼왕: 행사명 또는 행사 ID, 작품명, 출품자(회원명), 팀명, 평점

대체 수단:

- 자동 수집 실패 시 관리자는 엑셀 파일을 직접 업로드할 수 있다.
- 엑셀 업로드 시에도 행사 ID와 학교명/소속명/팀명 매칭 기준을 동일하게 적용한다.
- 수동 업로드 데이터는 자동 수집 데이터와 구분하여 저장한다.

알림:

- 연동 실패, 인증 실패, 페이지 구조 변경, 필수 컬럼 누락은 관리자 작업함에 표시한다.
- 중요한 실패는 관리자 이메일 또는 관리자 알림으로 전달할 수 있어야 한다.

## 6. 출품 데이터 자동 조회

시스템은 등록된 외부 서버 연동 정보를 기준으로 출품 데이터를 자동 조회한다.

자동 조회 시점:

- 하루 1회 정기 실행
- D-14, D-10, D-5, D-1 메일 발송 전 실행
- 관리자가 수동 동기화 버튼 클릭 시 실행

조회 대상:

- 해당 행사에 등록된 전체 출품 작품
- 또는 해당 행사에서 꿈프 학교명으로 검색된 작품

외부 서버에서 가져와야 하는 기본 데이터:

- 외부 행사 ID
- 외부 행사명
- 외부 출품 ID
- 작품명
- 출품 소속명 또는 팀명
- 행사 유형별 매칭명
  - 29초영화제: 출품 소속명
  - 29역숏폼왕: 팀명
- 꿈프 학교명
- 출품자명
- 출품일
- 작품 상세 URL
- 행사 ID 또는 행사 식별값
- 작품 상태

정책:

- 외부 서버에서 가져온 원본 데이터는 별도 테이블 또는 로그로 저장한다.
- 동일 외부 출품 ID는 중복 저장하지 않고 갱신한다.
- 외부 서버 응답 오류, 인증 오류, 데이터 형식 오류를 구분하여 기록한다.
- 외부 서버 연동 정보가 없는 경우 자동 매칭은 실행하지 않고 `연동 정보 미등록` 상태로 표시한다.
- 외부 행사 ID가 29 Platform 행사와 매핑되지 않으면 자동 매칭하지 않는다.
- 29역숏폼왕 출품 데이터에 팀명 값이 없으면 자동 매칭하지 않는다.

## 7. 자동 매칭 기준

자동 매칭은 꿈프 신청 시 저장된 학교명과 외부 서버의 출품 데이터에 있는 소속명 또는 팀명을 기준으로 수행한다.

기본 매칭 기준:

- 29 Platform 행사 ID와 외부 홈페이지 행사 ID 매핑
- 꿈프 신청의 `학교명` 값
- 29초영화제 외부 출품 데이터의 `소속명` 값
- 29역숏폼왕 외부 출품 데이터의 `팀명` 값

매칭 방식:

1. 행사 ID 일치
2. 꿈프 학교명과 외부 매칭값 완전 일치
3. 관리자 승인 기반 수동 매칭

완전 일치 기준:

- 띄어쓰기까지 동일해야 한다.
- `서울고등학교`와 `서울 고등학교`는 다른 값으로 본다.
- `서울고`와 `서울고등학교`는 다른 값으로 본다.
- 자동 매칭은 임의로 공백을 제거하거나 약칭을 추정하지 않는다.

자동 매칭 제외:

- 행사 ID가 확인되지 않는 경우
- 행사 ID가 일치하지 않는 경우
- 값이 비어 있는 경우
- 29역숏폼왕에서 팀명 값이 없는 경우
- 여러 학교 또는 여러 신청 건에 동일 학교명/소속명/팀명이 존재해 충돌하는 경우
- 외부 출품 작품 상태가 삭제 또는 취소인 경우

충돌 처리:

- 동일 학교명 또는 동일 외부 매칭값으로 여러 신청 건이 존재하면 자동 매칭하지 않는다.
- 관리자 화면에 `매칭 충돌` 상태로 표시한다.
- 관리자가 올바른 신청 건을 선택해 수동 매칭할 수 있다.

정책:

- 자동 매칭 결과는 이력을 남긴다.
- 수동 매칭과 자동 매칭을 구분하여 저장한다.
- 수동 매칭이 된 작품은 이후 자동 동기화에서 임의로 해제하지 않는다.

## 8. 출품 URL 자동 반영

자동 매칭된 작품은 작품 상세 URL을 꿈프 담당자 페이지와 관리자 화면에 표시한다.

표시 정보:

- 작품명
- 출품일
- 꿈프 학교명
- 외부 매칭값
  - 29초영화제 소속명
  - 29역숏폼왕 팀명
- 작품 상세 URL
- 매칭 방식
  - 자동
  - 수동
- 매칭 상태

매칭 상태:

- 매칭완료
- 미매칭
- 매칭충돌
- 연동정보미등록
- 연동실패
- 수동확인필요

정책:

- 자동으로 가져온 출품 URL은 담당자가 수정할 수 없다.
- 담당자가 직접 입력한 작품 링크와 자동 매칭 URL이 다를 경우 관리자 화면에 차이를 표시한다.
- 관리자는 수동으로 출품 URL을 추가하거나 수정할 수 있다.

## 9. 담당자 페이지

선정된 담당자 페이지에는 출품 진행 상황을 표시한다.

표시 정보:

- 행사명
- 출품 마감일
- 남은 기간
- 신청 작품 수
- 자동 확인된 출품 작품 수
- 작품 링크 입력란
- 자동 매칭된 출품 작품 목록
- 꿈프 학교명
- 29초영화제 소속명 또는 29역숏폼왕 팀명
- 출품 URL

상태 표시 예시:

- `신청 3편 / 출품 확인 0편`
- `신청 3편 / 출품 확인 2편`
- `신청 3편 / 출품 확인 3편`

안내 문구:

```text
출품 확인은 꿈나무양성프로젝트 신청 시 입력한 학교명을 기준으로 자동 확인됩니다.
29초영화제 출품 시 소속명, 29역숏폼왕 출품 시 팀명에 학교명을 띄어쓰기까지 동일하게 입력해야 자동 확인됩니다.
```

정책:

- 담당자는 자동 매칭된 작품 URL을 확인할 수 있다.
- 담당자는 자동 매칭되지 않은 경우 직접 작품 링크를 입력할 수 있다.
- 담당자가 직접 입력한 링크는 관리자 검수 대상이 된다.

## 10. 관리자 출품 진행 현황 화면

관리자는 행사별 꿈프 출품 진행 현황을 확인할 수 있다.

리스트 표시 항목:

- 학교명
- 담당교사명
- 29초영화제 소속명 또는 29역숏폼왕 팀명
- 신청 작품 수
- 자동 매칭 작품 수
- 직접 입력 작품 수
- 출품 마감일
- 남은 기간
- 매칭 상태
- 마지막 동기화 시각
- 메일 발송 상태

필터:

- 행사
- 매칭 상태
- 학교명
- 소속명/팀명
- 마감 D-day
- 자동 매칭 여부
- 수동 확인 필요 여부

관리자 액션:

- 수동 동기화
- 수동 매칭
- 매칭 해제
- 출품 URL 수정
- 안내 메일 재발송
- 엑셀 다운로드

## 11. 출품 완료 상태 처리

출품 완료는 신청 작품 수와 확인된 출품 작품 수를 기준으로 판단한다.

기본 기준:

- 자동 매칭 작품 수 + 관리자 승인 직접 입력 작품 수 >= 신청 작품 수

상태 처리:

- 기준을 충족하면 신청 상태를 `출품완료`로 변경할 수 있다.
- 자동 변경 또는 관리자 확인 후 변경 중 운영 정책을 선택할 수 있어야 한다.

권장 정책:

- 시스템은 `출품완료 가능` 상태를 자동 표시한다.
- 최종 `출품완료` 처리는 관리자가 확인 후 실행한다.

정책:

- 출품완료 처리 이력을 저장한다.
- 출품완료 이후에도 관리자 권한으로 매칭 정보를 수정할 수 있다.
- 출품완료 이후 매칭 작품 수가 부족해지면 관리자 화면에 경고를 표시한다.

## 12. 예외 상황

### 12.1 서버 정보 미등록

- 자동 조회를 실행하지 않는다.
- 관리자 화면에 `연동정보미등록`으로 표시한다.
- 담당자 페이지에는 직접 작품 링크 입력을 안내한다.

### 12.2 외부 서버 연결 실패

- 실패 사유를 저장한다.
- 관리자 화면에 `연동실패`로 표시한다.
- 다음 정기 실행 시 재시도한다.
- 관리자는 수동 동기화를 실행할 수 있다.

관리자 조치:

- 서버 접근 정보 수정
- 로그인 정보 재입력
- 수동 동기화 재실행
- 엑셀 파일 수동 업로드
- 해당 행사 자동 연동 일시 중지

### 12.3 학교명과 소속명/팀명 불일치

- 자동 매칭되지 않는다.
- 담당자 페이지에 학교명과 소속명/팀명 확인 안내를 표시한다.
- 관리자는 수동 매칭할 수 있다.

관리자 조치:

- 올바른 출품작 수동 매칭
- 학교명 변경 요청 승인
- 담당자에게 정정 요청 메일 발송
- 매칭 제외 처리

### 12.4 동일 학교명 또는 동일 소속명/팀명 충돌

- 자동 매칭하지 않는다.
- 관리자 화면에 `매칭충돌`로 표시한다.
- 관리자가 수동으로 올바른 신청 건에 연결한다.

관리자 조치:

- 후보 출품작 비교
- 올바른 신청 건 선택
- 수동 매칭
- 중복 신청 여부 표시

### 12.5 신청 작품 수보다 출품 작품 수가 많은 경우

- 초과 출품으로 표시한다.
- 자동 매칭은 유지한다.
- 관리자가 최종 지원 대상 작품 수를 확인한다.

### 12.6 신청 작품 수보다 출품 작품 수가 적은 경우

- 부족 출품으로 표시한다.
- D-day 안내 메일에 현재 확인된 작품 수를 포함한다.
- 마감 후에도 부족하면 후속 지원 대상 확정 단계에서 검토한다.

## 14. 개발 원칙

- 외부 서버 정보는 행사별로 분리 저장한다.
- 서버 정보가 없어도 담당자 페이지와 수동 링크 입력 기능은 동작해야 한다.
- 서버 정보가 나중에 입력되면 기존 선정 신청 건에 대해서도 자동 매칭을 실행할 수 있어야 한다.
- 자동 매칭은 꿈프 학교명과 29초영화제 소속명 또는 29역숏폼왕 팀명을 기준으로 한다.
- 자동 매칭이 불확실한 경우 임의 연결하지 않고 관리자 확인 대상으로 남긴다.
- 자동 처리 결과는 모두 이력을 남긴다.
- 담당자에게는 복잡한 연동 오류 대신 확인 가능한 출품 진행 상태만 보여준다.
- 관리자는 연동 오류와 매칭 충돌을 상세히 확인할 수 있어야 한다.
