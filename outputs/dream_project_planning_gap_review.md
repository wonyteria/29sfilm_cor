# 29 Platform Dream Project - 전체 기획 점검 및 보완 필요 사항

## 1. 현재 PRD 구성

현재 꿈나무양성프로젝트는 아래 4개 문서로 기능 흐름이 나뉘어 있다.

- 접수 및 선발
- 출품 진행 관리 및 자동 매칭
- 출품 확인, 활동확인서 생성 및 메일 발송
- 심사 결과 반영 및 심사표 PDF 다운로드

전체 흐름:

```text
회원가입
→ 행사 조회
→ 꿈프 신청
→ 선정
→ 출품 마감 전 안내
→ 외부 관리자 페이지 연동
→ 출품작 자동 매칭
→ 출품 마감 후 관리자 승인
→ 활동확인서 PDF 발급
→ 수상작 발표 후 심사 결과 반영
→ 심사표 PDF 다운로드
```

큰 흐름은 충분히 잡혀 있다. 다만 개발 착수 전 아래 항목은 추가 결정 또는 보강이 필요하다.

---

## 2. 가장 먼저 보완해야 할 항목

### 2.1 학교명과 출품 매칭명 정책 정리

현재 최신 기준은 아래와 같다.

- 꿈프 기준값: 학교명
- 29초영화제 매칭값: 소속명
- 29역숏폼왕 매칭값: 팀명
- 띄어쓰기까지 완전 일치해야 자동 매칭

보완 필요:

- 접수/선발 PRD에 남아 있는 `출품 소속명(팀명)` 필드명을 정리해야 한다.
- 신청 화면에서는 별도 출품 소속명을 받기보다, 학교명을 기준값으로 명확히 강조하는 것이 좋다.
- 기존 `출품 소속명 변경 승인` 표현은 `학교명 변경 승인` 또는 `출품 매칭명 확인`으로 바꾸는 것이 좋다.

권장:

```text
프로필 필수값: 학교명
안내값: 29초영화제 소속명/29역숏폼왕 팀명은 학교명과 동일 입력
```

### 2.2 학생 이름 데이터 수집 방식

활동확인서에는 학생 이름이 들어가야 한다.

확정 기준:

- 29초영화제: 출품작 관리 페이지의 `감독`
- 29역숏폼왕: 출품 관리 페이지의 `출품자(회원명)`

보완 완료:

- 활동확인서 PRD에 이름 데이터 출처를 반영했다.
- 감독명 또는 출품자명이 없는 경우 `이름정보누락`으로 관리자 작업함에 표시한다.

추가 확인 필요:

- `김세민(김세민)`처럼 회원명이 괄호로 중복 표시될 때 실제 활동확인서에는 어떤 값을 표시할지 최종 정책 확인이 필요하다.

### 2.3 활동확인서 템플릿 관리

템플릿 업로드 기능은 필요하다.

보완 완료:

- 활동확인서 PRD에 템플릿 업로드, 위치 설정, 테스트 발급, 버전 관리 추가.

추가 결정 필요:

- 템플릿 파일 형식: PDF만 허용할지, 이미지도 허용할지
- 기본 폰트
- 한 줄 초과 시 줄바꿈 규칙
- 작품 수가 많을 때 별첨 사용 여부

### 2.4 외부 관리자 페이지 연동 안정성

관리자 페이지 URL과 서버 접근 정보만으로 자동 수집이 가능하긴 하지만, 안정성을 위해 아래 정책이 더 필요하다.

보완 필요:

- 로그인 세션 만료 시 재로그인 방식
- 관리자 계정 비밀번호 변경 시 알림
- 페이지 구조 변경 감지
- 엑셀 다운로드 우선 사용 여부
- 수집 실패 시 재시도 횟수
- 수집 실패 알림 대상

권장:

```text
1순위: 엑셀 다운로드/API
2순위: 관리자 페이지 화면 수집
3순위: 관리자 수동 업로드
```

## 3. 추가로 기획하면 좋은 화면

### 3.1 관리자 통합 대시보드

행사별 운영 상태를 한눈에 볼 수 있는 화면이 있으면 좋다.

표시 항목:

- 신청 학교 수
- 선정 학교 수
- 신청 작품 수
- 확인된 출품 작품 수
- 출품 미확인 학교 수
- 활동확인서 발급 수
- 심사 결과 반영 수
- 메일 실패 수

### 3.2 담당자 마이페이지 타임라인

담당자가 현재 어디까지 진행됐는지 알기 쉽게 보여주는 화면이 필요하다.

예:

```text
신청 완료
→ 선정 완료
→ 출품 진행중
→ 출품 확인 완료
→ 활동확인서 발급 완료
→ 심사 결과 반영 완료
```

### 3.3 관리자 작업함

관리자가 처리해야 할 예외를 모아보는 화면이 필요하다.

예:

- 매칭 실패
- 팀명 필드 미등록
- 학생 이름 누락
- 활동확인서 생성 실패
- 심사 결과 매칭 실패
- 메일 발송 실패

보완 완료:

- 별도 PRD로 관리자 작업함 및 예외 조치 프로세스를 추가했다.
- 빠른 수동 조치를 기본으로 하고, AI는 원인 분석과 후보 추천을 보조하도록 정의했다.

## 4. 정책 결정이 필요한 항목

### 4.1 부분 출품 처리

신청 5편, 실제 출품 3편인 경우:

- 활동확인서를 발급할지
- 출품 완료로 볼지
- 미이행 횟수에 반영할지
- 향후 선정 우선순위에 어떻게 반영할지

### 4.2 초과 출품 처리

신청 3편, 실제 출품 5편인 경우:

- 활동확인서에 5편 모두 포함할지
- 신청 작품 수만큼만 포함할지
- 지원 대상 작품 수는 어떻게 볼지

### 4.3 예비/미선정 이후 재선정

예비자가 선정으로 바뀔 때:

- 쿠폰 지급 시점
- 출품 마감 안내 메일 포함 여부
- 이미 지난 D-day 메일 처리 방식

### 4.4 선정 취소

선정 취소 시:

- 쿠폰 회수 여부
- 담당자 페이지 비활성화 여부
- 출품작 리스트 유지 여부
- 활동확인서 발급 차단 여부

- 최종 승인 출품작 리스트
- 활동확인서
- 심사 결과 원본 데이터
- 심사표
- 쿠폰
- 메일 이력
- 수정 이력
- 외부 서버 연동 설정

## 6. 메일 기획 보완

현재 자동 메일 종류:

- 꿈프 모집 시작 안내
- 선정 안내
- 출품 마감 D-day 안내
- 출품 확인 완료 및 활동확인서 발급 안내
- 심사 결과 반영 안내

보완 필요:

- 메일 템플릿 관리 화면
- 메일 미리보기
- 발송 전 테스트 메일
- 수신 거부 또는 발송 제외 처리
- 메일 실패 재시도 정책
- 담당자 이메일 변경 시 과거 메일 이력 표시
