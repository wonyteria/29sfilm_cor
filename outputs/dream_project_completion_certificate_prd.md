# PRD - 29 Platform
## Dream Project(꿈나무양성프로젝트) - 출품 확인, 활동확인서 생성 및 메일 발송 프로세스

---

## 1. 프로젝트 개요

본 문서는 29초영화제 또는 29역숏폼왕 출품 마감 이후, 꿈나무양성프로젝트 활동이 실제로 수행되었는지 확인하고 활동확인서를 자동 생성하여 담당자에게 전달하는 기능을 정의한다.

핵심 목적:

- 출품 마감 후 담당자별 출품작 리스트 자동 생성
- 관리자 최종 확인 및 승인
- 승인 후 담당자에게 출품 확인 완료 안내 메일 자동 발송
- 활동확인서 PDF 자동 생성
- 담당자 마이페이지에서 활동확인서 다운로드 제공

본 단계는 출품 진행 관리 및 자동 매칭 프로세스 이후부터 활동확인서 발급까지의 운영을 다룬다.

---

## 2. 사용자 유형

### 2.1 관리자

29초영화제 사무국 담당자.

권한:

- 행사별 꿈프 출품작 리스트 조회
- 담당자별 출품작 리스트 확인
- 자동 매칭 결과 검토
- 출품작 수동 추가/제외
- 최종 리스트 승인
- 활동확인서 생성 상태 확인
- 활동확인서 PDF 다운로드
- 출품 확인 완료 메일 발송 및 재발송

### 2.2 꿈나무양성프로젝트 담당자

선정된 학교 선생님 또는 기관 담당자.

권한:

- 본인 학교의 출품작 리스트 확인
- 출품 확인 상태 확인
- 승인 후 활동확인서 PDF 다운로드
- 출품 확인 완료 메일 수신

---

## 3. 적용 대상

본 기능은 아래 조건을 만족하는 신청 건에 적용한다.

- 꿈나무양성프로젝트 신청 상태가 `선정`
- 출품 마감일이 지난 행사
- 출품 자동 매칭 또는 수동 매칭 결과가 존재하는 신청 건
- 관리자가 최종 검토 가능한 상태의 신청 건

---

## 4. 출품작 리스트 자동 생성

출품 마감 후 시스템은 담당자별 출품작 리스트를 자동 생성한다.

생성 기준:

- 29 Platform 행사 ID
- 외부 홈페이지 행사 ID
- 꿈프 학교명
- 29초영화제 출품 소속명
- 29역숏폼왕 팀명
- 자동 매칭된 출품작
- 관리자가 수동 매칭한 출품작

행사 유형별 매칭 기준:

- 29초영화제: 꿈프 학교명 = 출품작 소속명
- 29역숏폼왕: 꿈프 학교명 = 출품작 팀명

완전 일치 기준:

- 띄어쓰기까지 동일해야 한다.
- 약칭 또는 유사 학교명은 자동 매칭하지 않는다.
- 자동 매칭되지 않은 경우 관리자 수동 확인 대상으로 남긴다.

출품작 리스트 표시 항목:

- 행사명
- 행사 유형
- 학교명
- 담당교사명
- 담당자 연락처
- 담당자 이메일
- 신청 작품 수
- 확인된 출품 작품 수
- 작품명
- 이름
  - 29초영화제: 감독
  - 29역숏폼왕: 출품자(회원명)
- 출품일
- 출품 소속명 또는 팀명
- 작품 URL
- 매칭 방식
  - 자동
  - 수동
- 매칭 상태
- 최종 승인 상태

정책:

- 출품작 리스트는 담당자와 관리자 모두 확인할 수 있다.
- 담당자는 본인 신청 건의 출품작 리스트만 확인할 수 있다.
- 관리자는 전체 담당자의 출품작 리스트를 확인할 수 있다.
- 출품작이 0건인 경우에도 리스트를 생성하고 `출품작 없음` 상태로 표시한다.

## 5. 관리자 최종 확인 및 승인

출품 마감 후 관리자는 담당자별 출품작 리스트를 최종 확인한다.

관리자 액션:

- 출품작 리스트 확인
- 출품작 수동 추가
- 출품작 제외
- 작품 URL 수정
- 메모 작성
- 최종 승인

승인 버튼:

- 담당자별 출품작 리스트에 `승인` 버튼을 제공한다.
- 승인 전에는 활동확인서 다운로드가 비활성화된다.
- 승인 후 활동확인서 PDF가 생성되고 담당자에게 안내 메일이 발송된다.

승인 조건:

- 출품 마감일이 지난 상태
- 출품작 리스트가 생성된 상태
- 관리자가 최종 확인한 상태

승인 후 자동 처리:

1. 출품작 리스트 승인 상태 변경
2. 신청 상태를 `출품완료`로 변경
3. 활동확인서 PDF 생성
4. 담당자 페이지의 활동확인서 다운로드 버튼 활성화
5. 출품 확인 완료 안내 메일 발송
6. 승인 이력 및 메일 발송 이력 저장

정책:

- 승인 후에도 관리자는 필요 시 승인 취소할 수 있다.
- 승인 취소 시 활동확인서 다운로드를 비활성화한다.
- 승인 취소 이력을 저장한다.
- 승인 후 출품작 리스트가 변경되면 기존 활동확인서는 재생성 대상이 된다.

## 6. 담당자 페이지

담당자 마이페이지에는 출품 확인 상태와 출품작 리스트를 표시한다.

표시 정보:

- 행사명
- 행사 유형
- 학교명
- 신청 작품 수
- 확인된 출품 작품 수
- 최종 승인 상태
- 활동확인서 다운로드 버튼
- 출품작 리스트

출품작 리스트 항목:

- 작품명
- 출품일
- 작품 URL
- 출품 소속명 또는 팀명
- 매칭 상태

활동확인서 버튼 상태:

- 승인 전: 비활성화
- 승인 후: 활성화
- 승인 취소: 비활성화

비활성화 문구:

```text
관리자 최종 확인 후 활동확인서를 다운로드할 수 있습니다.
```

활성화 문구:

```text
활동확인서 다운로드
```

## 7. 활동확인서 자동 생성

관리자가 출품작 리스트를 승인하면 활동확인서를 자동 생성한다.

파일 형식:

- PDF

생성 단위:

- 담당자 신청 건 단위
- 하나의 행사에 대해 하나의 학교/담당자별 활동확인서 생성

템플릿:

- 기본 활동확인서 템플릿을 사용한다.
- 실제 발급본에는 화면 예시의 빨간 네모 표시를 포함하지 않는다.
- 템플릿 디자인은 고정하고, 지정된 영역의 텍스트만 데이터로 치환한다.
- 관리자는 활동확인서 템플릿을 직접 업로드할 수 있다.
- 템플릿은 PDF 업로드를 우선 지원하고, 필요 시 PNG/JPG 이미지 업로드를 보조 지원한다.

### 7.1 템플릿 관리

관리자는 활동확인서 템플릿을 등록하고 관리할 수 있다.

관리 기능:

- 템플릿 업로드
- 템플릿 미리보기
- 텍스트 필드 위치 설정
- 텍스트 스타일 설정
- 테스트 발급
- 사용 여부 설정
- 기본 템플릿 지정
- 템플릿 버전 관리

텍스트 필드 설정 항목:

- 필드명
- 표시 위치
- 글꼴
- 글자 크기
- 굵기
- 정렬
- 줄바꿈 여부
- 최대 너비
- 최대 줄 수

위치 설정 대상 필드:

- 확인서 번호
- 학교명
- 작품명
- 이름
- 활동 기간
- 발급일
- 주관 문구

정책:

- 템플릿 업로드만으로 자동 발급하지 않고, 최초 1회 텍스트 위치 설정을 완료해야 사용할 수 있다.
- 템플릿 위치 설정이 완료되지 않은 경우 활동확인서 생성 상태를 `템플릿설정필요`로 표시한다.
- 기존 발급 PDF는 템플릿이 변경되어도 자동 변경하지 않는다.
- 템플릿 변경 후 신규 발급 또는 재발급 시 변경된 템플릿을 적용한다.

자동 입력 항목:

- 확인서 번호
- 학교명
- 작품명
- 이름
- 활동 기간
- 발급일
- 주관 문구
- 행사명 또는 프로젝트명

항목별 데이터 기준:

### 7.2 확인서 번호

형식 예시:

```text
제 2602 호
```

정책:

- 확인서 번호는 시스템에서 자동 발급한다.
- 중복 발급되지 않아야 한다.
- 행사별 또는 연도별 일련번호 정책을 설정할 수 있어야 한다.
- 재발급 시 기존 번호 유지 또는 재발급 번호 생성 정책을 관리자가 선택할 수 있어야 한다.

### 7.3 학교명

데이터 출처:

- 꿈프 프로필의 학교명

예:

```text
학교: 방학중학교
```

### 7.4 작품명

데이터 출처:

- 승인된 출품작 리스트의 작품명

정책:

- 작품이 1개인 경우 작품명 그대로 표시한다.
- 작품이 여러 개인 경우 쉼표로 구분하여 표시한다.
- 작품명이 너무 길거나 작품 수가 많아 한 줄에 표시하기 어려운 경우 줄바꿈 처리한다.
- 필요 시 별첨 목록 형태를 사용할 수 있어야 한다.

예:

```text
작품명: 똑같은 고민, 달라진 속도
```

### 7.5 이름

데이터 출처:

- 29초영화제: 출품작 관리 페이지의 `감독`
- 29역숏폼왕: 출품 관리 페이지의 `출품자(회원명)`

정책:

- 29초영화제 활동확인서의 이름은 외부 출품 데이터의 감독명을 사용한다.
- 29역숏폼왕 활동확인서의 이름은 외부 출품 데이터의 출품자명을 사용한다.
- 이름이 여러 명이면 쉼표로 구분하여 표시한다.
- 감독명 또는 출품자명이 비어 있으면 관리자 입력 또는 수동 보완이 가능해야 한다.
- 담당교사명은 학생 이름을 대체하지 않는다.
- 회원명 괄호 표기가 있는 경우 표시명 추출 정책을 설정할 수 있어야 한다.
  - 예: `김세민(김세민)` → 기본 표시값 `김세민`
  - 원본 값은 별도 저장한다.

예:

```text
이름: 강하은, 김희야, 노유림, 이다은
```

### 7.6 활동 기간

데이터 출처:

- 행사 공모기간
- 또는 꿈프 활동 기간이 별도 등록되어 있으면 해당 기간 우선 사용

예:

```text
2026년 4월 8일부터 2026년 5월 21일까지
```

정책:

- 활동 기간은 관리자가 행사별로 수정할 수 있어야 한다.
- 기본값은 행사 공모기간으로 한다.

### 7.7 발급일

데이터 출처:

- 관리자 승인일
- 또는 관리자가 지정한 발급일

예:

```text
2026. 7. 1.
```

정책:

- 기본값은 승인일이다.
- 관리자는 발급일을 수정할 수 있다.

## 8. 활동확인서 문구

기본 문구:

```text
위 학생은 {활동시작일}부터 {활동종료일}까지
29초영화제사무국이 주관한
<영상 꿈나무 양성 프로젝트>에서 활동하며,
맡은 역할을 열정적인 자세로
성실히 수행하였기에 이 증서를 수여합니다.
```

정책:

- 행사 유형과 관계없이 기본 문구는 `영상 꿈나무 양성 프로젝트`를 사용한다.
- 주관 문구는 기본적으로 `29초영화제사무국`을 사용한다.
- 문구는 관리자 설정에서 템플릿별로 수정 가능해야 한다.

## 9. 활동확인서 다운로드

활동확인서는 PDF 형태로 다운로드한다.

다운로드 제공 위치:

- 담당자 마이페이지
- 관리자 출품작 리스트 상세

다운로드 가능 시점:

- 출품 마감 후
- 관리자가 최종 출품작 리스트를 승인한 후
- PDF 생성이 완료된 후

정책:

- 승인 전에는 다운로드할 수 없다.
- PDF 생성 실패 시 다운로드 버튼을 비활성화하고 관리자에게 오류를 표시한다.
- 담당자는 본인의 활동확인서만 다운로드할 수 있다.
- 관리자는 전체 활동확인서를 다운로드할 수 있다.
- 다운로드 이력을 저장한다.

---

## 10. 출품 확인 완료 메일

관리자가 출품작 리스트를 승인하면 담당자에게 자동 메일을 발송한다.

메일 발송 시점:

- 관리자 승인 직후

메일 제목 예시:

```text
[29 Platform] 영상 꿈나무 양성 프로젝트 출품 확인 및 활동확인서 발급 안내
```

메일 포함 내용:

- 담당자명
- 학교명
- 행사명
- 행사 유형
- 확인된 출품 작품 수
- 출품작 리스트 요약
- 활동확인서 다운로드 가능 안내
- 담당자 페이지 바로가기

메일 문구 예시:

```text
안녕하세요. 29초영화제사무국입니다.

{행사명} 영상 꿈나무 양성 프로젝트 출품 확인이 완료되었습니다.

학교명: {학교명}
확인된 출품 작품 수: {확인작품수}편

담당자 페이지에서 출품작 리스트를 확인하고
활동확인서를 PDF로 다운로드하실 수 있습니다.
```

정책:

- 메일 발송 성공/실패 이력을 저장한다.
- 실패한 메일은 관리자 화면에서 재발송할 수 있다.
- 재발송 이력을 저장한다.
- 승인 취소 후에는 자동 메일을 재발송하지 않는다.

## 13. 예외 상황

### 13.1 출품작이 없는 경우

- 담당자별 리스트에 `출품작 없음`으로 표시한다.
- 관리자는 승인할 수 없다.
- 활동확인서는 생성하지 않는다.

### 13.2 신청 작품 수보다 출품 확인 작품 수가 적은 경우

- `일부확인` 상태로 표시한다.
- 관리자는 운영 판단에 따라 승인할 수 있다.
- 승인 시 실제 확인된 작품 수 기준으로 메일과 활동확인서를 생성한다.

### 13.3 신청 작품 수보다 출품 확인 작품 수가 많은 경우

- 초과 출품으로 표시한다.
- 관리자는 활동확인서에 포함할 작품을 선택할 수 있다.

### 13.4 학생 이름이 없는 경우

- 활동확인서 생성 전 관리자 입력 또는 보완이 필요하다.
- 29초영화제 감독명 또는 29역숏폼왕 출품자명이 없으면 활동확인서 상태를 `생성대기`로 유지한다.
- 관리자 작업함에 `이름정보누락`으로 표시한다.

### 13.5 PDF 생성 실패

- 관리자 화면에 `생성실패`로 표시한다.
- 실패 사유를 저장한다.
- 관리자는 재생성을 실행할 수 있다.

## 15. 개발 원칙

- 활동확인서는 관리자 승인 이후에만 생성한다.
- 승인 전에는 담당자에게 활동확인서 다운로드를 제공하지 않는다.
- 자동 매칭 결과를 그대로 확정하지 않고 관리자 최종 승인 단계를 반드시 둔다.
- 활동확인서에 들어가는 데이터는 승인된 출품작 리스트를 기준으로 한다.
- PDF 생성 결과와 메일 발송 결과는 각각 독립적으로 이력을 남긴다.
- 담당자는 본인 학교의 출품작 리스트와 활동확인서만 볼 수 있다.
- 관리자는 모든 학교의 출품작 리스트와 활동확인서를 관리할 수 있다.
