# 29 WITH - 꿈프 페이지별 화면 구성 및 기능 설명서

## 1. 문서 목적

이 문서는 꿈나무양성프로젝트를 직접 운영할 관리자가 29 WITH의 각 페이지가 어떤 역할을 하는지 이해할 수 있도록 정리한 화면별 설명서다.

개발 전 확인 목적:

- 각 페이지가 왜 필요한지 확인
- 관리자가 어느 화면에서 어떤 업무를 하는지 확인
- 버튼을 누르면 어떤 일이 실행되는지 확인
- 여러 꿈프가 동시에 진행될 때 헷갈리지 않는 구조인지 확인
- 선생님 화면과 관리자 화면이 어떻게 연결되는지 확인

이 문서는 기술 문서가 아니라 운영자 이해를 위한 문서다.

---

## 2. 29 WITH 전체 구조

29 WITH는 한국경제신문 29 Platform의 대외협력 관리 시스템이다.

상단 카테고리:

```text
전체
꿈프
서포터즈
명예홍보대사
파트너사
```

현재 개발 범위:

```text
꿈프
```

운영 원칙:

- 여러 꿈프가 동시에 진행될 수 있다.
- 같은 학교가 여러 꿈프에 동시에 참여할 수 있다.
- 같은 학교의 담당 선생님이 2명 이상 신청할 수 있으며, 이 경우 관리자가 중복 가능성을 확인한다.
- 선정은 기본적으로 선착순이지만 성실 참여 이력과 미이행 이력에 따라 순위가 조정될 수 있다.
- 따라서 모든 관리 화면은 반드시 “현재 선택한 꿈프 행사” 기준으로 보여야 한다.

관리자 화면 기본 흐름:

```text
전체 대시보드
→ 꿈프 선택
→ 진행 중인 꿈프 행사 선택
→ 선택한 행사 기준으로 신청/선정, 출품 확인, 확인서, 심사표 관리
```

## 3. 관리자 페이지 전체 메뉴

꿈프 관리자 메뉴:

```text
꿈프 대시보드
행사 운영
회원/참여 학교
신청/선정
출품 확인
혜택/지원
활동확인서
심사 결과
메일/공지
문의/요청
연동 설정
히스토리
```

관리자는 먼저 `꿈프 대시보드`에서 진행 중인 꿈프를 선택한다.

꿈프를 선택하면 모든 메뉴는 해당 행사 기준으로 바뀐다.

예시:

```text
현재 선택된 꿈프
제13회 박카스 29초영화제
29초영화제 · 모집중 · 출품 마감 D-5
```

## 4. 전체 대시보드

### 4.1 페이지 목적

29 WITH 전체 운영 상태를 보는 첫 화면이다.

여기서는 꿈프뿐 아니라 추후 서포터즈, 명예홍보대사, 파트너사까지 한눈에 볼 수 있다.

### 4.2 화면 구성

표시 정보:

- 오늘 처리해야 할 전체 업무 수
- 꿈프 처리 필요 건수
- 서포터즈 처리 필요 건수
- 명예홍보대사 처리 필요 건수
- 파트너사 처리 필요 건수
- 운영 중인 프로젝트 수

### 4.3 주요 버튼

`꿈프 보기`

- 꿈프 대시보드로 이동한다.

### 4.4 운영자에게 중요한 점

전체 대시보드는 상세 업무를 처리하는 곳이 아니다.

오늘 어떤 카테고리에 처리할 일이 있는지 확인하고 해당 카테고리로 들어가는 화면이다.

---

## 5. 꿈프 대시보드

### 5.1 페이지 목적

현재 진행 중인 꿈프 행사를 한눈에 확인하고, 가장 먼저 처리해야 할 행사를 선택하는 화면이다.

### 5.2 화면 구성

진행 중인 꿈프 카드:

- 행사명
- 행사 유형
- 모집/출품/마감 상태
- 선정 학교 수
- 신청 작품 수
- 출품 확인 수
- 미확인 작품 수
- 처리 필요 건수
- 출품 마감 D-day

예시:

```text
제13회 박카스 29초영화제
선정 28교 · 출품 확인 104/122편 · 처리 필요 6건
출품 마감 D-5
```

오늘 할 일:

- 미확인 작품
- 메일 재발송 필요
- 간식비 예정일 미입력
- 활동확인서 승인 대기
- 문의/요청 미처리

### 5.3 주요 버튼

`행사 보기`

- 해당 꿈프의 행사 운영 화면으로 이동한다.

`출품 확인`

- 해당 꿈프의 출품 확인 화면으로 이동한다.

`처리 필요 보기`

- 해당 꿈프에서 처리해야 할 항목만 모아서 보여준다.

### 5.4 운영자에게 중요한 점

여러 꿈프가 동시에 진행될 수 있으므로, 이 화면에서 어떤 꿈프를 관리할지 먼저 선택해야 한다.

선택하지 않은 상태에서는 신청/선정, 출품 확인, 활동확인서 같은 세부 업무를 진행하지 않는다.

---

## 6. 행사 운영

### 6.1 페이지 목적

선택한 꿈프 행사의 전체 운영 단계를 확인하고, 현재 단계에서 해야 할 일을 처리하는 화면이다.

### 6.2 화면 구성

상단 고정 정보:

- 행사명
- 행사 유형
- 공모기간
- 출품 마감일
- 꿈프 운영 상태
- 외부 연동 상태

운영 단계:

```text
1. 행사 등록
2. 꿈프 모집 시작
3. 신청/선정
4. 출품 자동 확인
5. 최종 리스트 승인
6. 활동확인서 발급
7. 심사표 발급
8. 종료
```

각 단계는 아래 상태로 표시된다.

- 완료
- 진행 중
- 확인 필요
- 대기

### 6.3 주요 버튼

`행사 정보 수정`

- 행사명, 공모기간, 홈페이지 URL, 출품 URL, 안내사항 등을 수정한다.

`꿈프 모집 시작`

- 신청 버튼을 활성화한다.
- 꿈프 회원에게 모집 안내 메일을 예약 또는 발송한다.
- 행사 상태를 모집중으로 변경한다.

`신청자 보기`

- 신청/선정 화면으로 이동한다.

`출품 확인`

- 출품 확인 화면으로 이동한다.

`최종 리스트 승인`

- 출품 마감 후 학교별 최종 출품작 리스트 승인 화면으로 이동한다.

`꿈프 종료`

- 모든 절차가 끝난 뒤 꿈프를 종료하고 히스토리로 이동한다.

### 6.4 팝업이 필요한 경우

꿈프 모집 시작:

```text
꿈프 모집을 시작하시겠습니까?
실행 시 신청 버튼이 활성화되고 모집 안내 메일이 발송됩니다.
```

모집 중 행사 정보 수정:

```text
모집 중인 행사 정보를 수정하시겠습니까?
예약 메일, 선생님 안내, 출품 확인 일정이 달라질 수 있습니다.
```

꿈프 종료:

```text
꿈프를 종료하시겠습니까?
종료 후 진행 중 목록에서 사라지고 히스토리에 보관됩니다.
```

### 6.5 운영자에게 중요한 점

행사 운영 화면은 “전체 흐름 확인” 화면이다.

실제 세부 업무는 신청/선정, 출품 확인, 활동확인서, 심사 결과 같은 각 메뉴에서 처리한다.

## 7. 회원/참여 학교

### 7.1 페이지 목적

꿈프 회원으로 가입한 선생님과 학교 정보를 확인하고, 학교별 참여 이력과 관리자 메모를 관리하는 화면이다.

### 7.2 화면 구성

학교 리스트:

- 학교명
- 담당교사
- 연락처
- 이메일
- 최근 참여 상태
- 신청 횟수
- 선정 횟수
- 출품 완료 횟수
- 미이행 횟수

학교 상세:

- 학교 프로필
- 담당교사 정보
- 출품 소속명/팀명
- 통장사본 업로드 상태
- 신분증 업로드 상태
- 참여 이력
- 관리자 메모

### 7.3 주요 버튼

`프로필 보기`

- 학교와 담당교사 정보를 확인한다.

`참여 이력 보기`

- 과거 신청, 선정, 출품 완료, 미이행 이력을 확인한다.

`메모 추가`

- 학교 단위 관리자 메모를 남긴다.

`학교명 변경 승인`

- 선생님이 요청한 학교명 변경을 승인하거나 반려한다.

`메일 보내기`

- 해당 학교 담당자에게 메일을 보낸다.

### 7.4 운영자에게 중요한 점

관리자 메모는 행사별이 아니라 학교 단위로 누적된다.

예시:

- 담당교사 응답 빠름
- 학생 퀄리티 우수
- 지난 행사 미이행
- 연락 완료

---

## 8. 신청/선정

### 8.1 페이지 목적

선생님이 신청한 학교를 검토하고 선정, 예비, 미선정 처리하는 화면이다.

### 8.2 화면 구성

상단 요약:

- 신청 학교 수
- 목표 선정 학교 수
- 예상 출품 작품 수
- 모집 목표 작품 수
- 목표 초과 여부

필터:

- 전체
- 검토 필요
- 성실 참여
- 확인 필요 이력
- 선정
- 예비
- 미선정

학교 리스트:

- 신청일
- 학교명
- 담당교사
- 연락처
- 이메일
- 출품 소속명/팀명
- 예상 출품편수
- 출품 예정일
- 참여 신뢰 상태
- 신청 순번
- 선정 추천 순위
- 신청 상태

상세 패널:

- 신청서 내용
- 활용 계획
- 메모
- 참여 이력
- 관리자 메모

### 8.3 주요 버튼

`선정 처리`

선정 처리 시 자동 실행:

- 신청 상태를 선정으로 변경
- 선정 안내 메일 예약 또는 발송
- 구독권 쿠폰 지급
- 선생님 담당자 페이지 생성
- 신청 작품 수만큼 작품 슬롯 생성
- 선생님 대시보드에 선정 완료 표시
- 작업 이력 저장

선정 기준:

- 기본은 선착순이다.
- 성실 참여 학교/선생님은 우선 검토될 수 있다.
- 미이행 이력이 있는 학교/선생님은 선착순 순위에서 뒤로 밀릴 수 있다.
- 활동 우수 학교 선정 시 성실 참여 이력은 유리하게 반영된다.

`예비 처리`

- 신청 상태를 예비로 변경한다.
- 예비 안내 메일을 예약 또는 발송한다.

`미선정 처리`

- 신청 상태를 미선정으로 변경한다.
- 미선정 안내 메일을 예약 또는 발송한다.

`메일 미리보기`

- 선정/예비/미선정 메일 내용을 발송 전 확인한다.

`엑셀 다운로드`

- 신청자 목록을 엑셀로 다운로드한다.

### 8.4 팝업이 필요한 경우

선정 처리:

```text
선택한 학교를 선정 처리하시겠습니까?
선정 메일, 구독권 지급, 담당자 페이지 생성이 함께 진행됩니다.
```

쿠폰 부족:

```text
구독권 쿠폰 수량이 부족합니다.
선정은 가능하지만 일부 학교는 구독권 지급 예정 상태로 표시됩니다.
```

중복 신청 가능성:

```text
중복 신청 가능성이 있는 학교가 있습니다.
선정 전 신청 정보를 확인해 주세요.
```

### 8.5 운영자에게 중요한 점

선정 처리는 단순히 상태만 바꾸는 버튼이 아니다.

선정 처리 후 선생님 화면과 혜택, 작품 관리 화면이 함께 생성되므로 신중하게 처리해야 한다.

같은 학교에서 여러 명이 신청한 경우 자동 병합하지 말고, 담당자에게 연락해 1명으로 정리하거나 출품 소속명/팀명에 A, B 등 구분값을 붙이도록 안내한다.

## 9. 출품 확인

### 9.1 페이지 목적

외부 29초영화제/29역숏폼왕 관리자 페이지에서 가져온 출품작을 꿈프 신청 학교와 매칭하고, 학교별 출품 현황을 확인하는 화면이다.

### 9.2 화면 구성

상단 요약:

- 선정 학교 수
- 신청 작품 수
- 자동 확인 작품 수
- 수동 확인 작품 수
- 미확인 작품 수
- 초과 출품 수
- 마지막 동기화 시간

필터:

- 전체
- 미확인 있음
- 자동 확인 완료
- 수동 확인 필요
- 초과 출품
- 선생님 확인 요청 중

학교 리스트:

- 학교명
- 출품 소속명/팀명
- 신청 작품 수
- 확인 작품 수
- 미확인 작품 수
- 초과 출품 수
- 매칭 상태
- 마지막 확인 시간

학교 상세:

- 작품명
- 감독명 또는 출품자명
- 출품 URL
- 출품일
- 자동/수동 매칭 여부
- 확인 상태

### 9.3 주요 버튼

`전체 동기화`

- 선택한 행사 기준으로 외부 출품작 데이터를 가져온다.

`선택 학교 재확인`

- 특정 학교만 다시 출품 확인한다.

`수동 매칭`

- 자동 매칭되지 않은 작품을 관리자가 직접 학교에 연결한다.

`선생님에게 확인 요청`

- 미확인 작품이 있는 경우 선생님에게 확인 요청을 보낸다.

`작품 링크 추가`

- 관리자가 직접 작품 링크를 추가한다.

### 9.4 중요한 매칭 기준

자동 매칭은 반드시 아래 기준으로 한다.

```text
꿈프 행사 ID + 외부 행사 ID + 참여 ID + 출품 소속명/팀명 완전 일치
```

29초영화제:

```text
출품 소속명 = 외부 출품작의 소속명
```

29역숏폼왕:

```text
출품 소속명 = 외부 출품작의 팀명
```

학교명만으로 매칭하면 안 된다.

외부 행사 식별 기준:

- 행사명은 사람이 확인하는 표시값이다.
- 행사명만 고유 ID로 사용하지 않는다.
- 외부 행사 ID가 없으면 29 WITH에서 행사별 연결 ID를 생성해 사용한다.
- 외부 관리자 페이지 URL, 행사명, 연도, 행사 유형, 연결 ID를 함께 관리한다.

### 9.5 팝업이 필요한 경우

수동 매칭:

```text
선택한 출품작을 이 학교에 연결하시겠습니까?
잘못 연결하면 활동확인서와 심사표에 영향을 줄 수 있습니다.
```

이미 다른 학교에 연결된 작품:

```text
이미 다른 학교에 연결된 작품입니다.
연결을 변경하시겠습니까?
```

동기화 결과 급변:

```text
이전 동기화 결과와 큰 차이가 있습니다.
외부 데이터 또는 연동 설정을 확인한 후 반영해 주세요.
```

### 9.6 운영자에게 중요한 점

첫 화면에서는 작품 전체 리스트가 아니라 학교 단위 요약을 먼저 봐야 한다.

학교당 작품 수가 많기 때문에 전체 작품을 한 번에 펼치면 관리가 어려워진다.

## 10. 혜택/지원

### 10.1 페이지 목적

선정된 학교에 제공되는 구독권, 간식비, 기타 지원 상태를 관리하는 화면이다.

### 10.2 화면 구성

상단 요약:

- 구독권 지급 완료 학교 수
- 쿠폰 미지급 학교 수
- 간식비 지급 예정일 입력 학교 수
- 간식비 예정일 미입력 학교 수
- 보류 학교 수

학교 리스트:

- 학교명
- 담당교사
- 구독권 상태
- 쿠폰번호
- 간식비 상태
- 지급 예정일
- 지급 금액
- 선생님 노출 문구

### 10.3 주요 버튼

`쿠폰 업로드`

- 쿠폰 엑셀 파일을 업로드한다.

`쿠폰 지급`

- 선택한 학교에 미사용 쿠폰을 지급한다.

`간식비 지급 예정일 입력`

- 선생님 화면에 표시될 지급 예정일을 입력한다.

`지급 완료 처리`

- 간식비 지급 완료 상태로 변경한다.

`보류 사유 입력`

- 간식비 지급 보류 사유를 입력한다.

`지원 안내 메일 발송`

- 간식비 또는 지원 관련 안내 메일을 발송한다.

### 10.4 운영자 입력 항목

간식비:

- 지급 상태
- 지급 예정일
- 지급 금액
- 관리자 내부 사유
- 선생님 노출 문구

주의:

- 관리자 내부 사유와 선생님에게 보이는 문구는 분리한다.

예시:

```text
관리자 내부 사유: 통장사본 식별 불가
선생님 노출 문구: 간식비 지급을 위해 증빙 확인이 필요합니다.
```

### 10.5 팝업이 필요한 경우

간식비 일괄 변경:

```text
선택한 학교의 간식비 지급 예정일을 변경하시겠습니까?
변경된 예정일은 선생님 대시보드에 표시됩니다.
```

쿠폰 중복 지급:

```text
이미 구독권 쿠폰이 지급된 학교입니다.
추가 지급하시겠습니까?
```

### 10.6 운영자에게 중요한 점

선생님은 간식비를 “언제 받을 수 있는지”를 가장 궁금해한다.

따라서 지급 예정일이 정해지면 반드시 입력해야 한다.

## 11. 활동확인서

### 11.1 페이지 목적

출품 마감 후 학교별 최종 출품작 리스트를 승인하고, 활동확인서 PDF를 생성/발송하는 화면이다.

### 11.2 화면 구성

상단 요약:

- 승인 대기 학교 수
- PDF 생성 필요 학교 수
- 발급 완료 학교 수
- 재발급 요청 수

필터:

- 승인 대기
- PDF 생성 필요
- 발급 완료
- 재발급 요청

학교 리스트:

- 학교명
- 최종 확인 작품 수
- 학생 이름 확인 여부
- 승인 상태
- 확인서 상태

상세:

- 작품명
- 학생 이름
- 활동 기간
- 발급 예정일
- PDF 미리보기

### 11.3 주요 버튼

`최종 승인`

- 해당 학교의 최종 출품작 리스트를 확정한다.
- 선생님 화면에 출품 확인 완료 상태를 표시한다.
- 활동확인서 PDF 생성 가능 상태로 변경한다.

`PDF 미리보기`

- 실제 생성 전 확인서 내용을 미리 확인한다.

`활동확인서 PDF 생성`

- PDF 파일을 생성한다.

`발급 메일 발송`

- 선생님에게 활동확인서 다운로드 안내 메일을 보낸다.

`재발급`

- 수정된 내용으로 활동확인서를 다시 발급한다.

### 11.4 버튼 노출 조건

- 출품 마감 전에는 최종 승인 버튼 비활성
- 학생 이름 누락 시 PDF 생성 버튼 비활성
- 최종 승인 전에는 PDF 다운로드 불가
- PDF 생성 후에만 발급 메일 발송 가능

### 11.5 팝업이 필요한 경우

최종 승인:

```text
최종 출품 리스트를 승인하시겠습니까?
승인 후 활동확인서 PDF 생성이 가능해집니다.
```

재발급:

```text
활동확인서를 재발급하시겠습니까?
기존 발급본과 재발급 이력이 함께 저장됩니다.
```

### 11.6 운영자에게 중요한 점

활동확인서는 선생님과 학교가 실제로 다운로드하는 공식 문서다.

PDF 생성 전 학교명, 작품명, 학생 이름, 활동 기간을 반드시 미리보기로 확인해야 한다.

## 12. 심사 결과

### 12.1 페이지 목적

수상작 발표 후 외부 관리자 페이지의 심사 결과를 가져와 학교별 순위와 본심 진출 여부를 반영하고, 심사표 PDF를 생성하는 화면이다.

### 12.2 화면 구성

상단 요약:

- 결과 반영 완료 학교 수
- 반영 필요 학교 수
- 심사표 생성 완료 수
- 안내 메일 발송 수

학교 리스트:

- 학교명
- 작품 수
- 순위 반영 상태
- 본심 반영 상태
- 심사표 상태

작품 상세:

- 작품명
- 감독명/출품자명
- 순위
- 본심 진출 여부

### 12.3 주요 버튼

`심사 결과 동기화`

- 외부 관리자 페이지에서 심사 결과를 가져온다.

`결과 검토`

- 반영 전 데이터를 검토한다.

`심사표 PDF 생성`

- 학교별 심사표 PDF를 생성한다.

`결과 안내 메일 발송`

- 선생님에게 심사표 안내 메일을 보낸다.

### 12.4 행사 유형별 차이

29초영화제:

- 예심점수 표시
- 순위 표시
- 본심 진출 여부 표시

29역숏폼왕:

- 평점 표시
- 순위 표시
- 본심 진출 여부는 표시하지 않음

순위 기준:

- 같은 소속명/팀명 출품작 리스트 안에서 예심평점 또는 평점 기준으로 정렬한다.
- 29초영화제 본심 여부는 수상결과가 `1차통과`인 경우 `진출`로 표시한다.

### 12.5 팝업이 필요한 경우

심사 결과 급변:

```text
기존 심사 결과와 차이가 큰 데이터가 감지되었습니다.
반영 전 결과를 검토해 주세요.
```

결과 안내 메일 발송:

```text
심사 결과 안내 메일을 발송하시겠습니까?
발송 후 선생님이 심사표를 확인할 수 있습니다.
```

### 12.6 운영자에게 중요한 점

선생님에게도 예심점수 또는 평점은 보여준다.

다만 원본 심사 내부값이나 관리자 내부 데이터는 보여주지 않는다.

## 13. 메일/공지

### 13.1 페이지 목적

자동 메일과 수동 공지를 관리하는 화면이다.

메일 내용, 발송일, 수신자 이메일을 발송 전 확인하고 수정할 수 있다.

발신자 주소도 관리자가 수정하고 관리할 수 있어야 한다.

### 13.2 화면 구성

탭:

```text
템플릿
예약 메일
발송 이력
실패/재발송
수신자 목록
발신자 설정
```

### 13.3 템플릿 탭

목적:

- 자동 메일의 기본 제목과 본문을 관리한다.

표시:

- 메일 유형
- 기본 제목
- 기본 본문
- 사용 변수
- 마지막 수정일
- 수정자

주요 버튼:

- `템플릿 수정`
- `미리보기`
- `테스트 발송`
- `기본값 복원`

### 13.4 예약 메일 탭

목적:

- 앞으로 발송될 메일을 미리 확인하고 수정한다.

표시:

- 메일 유형
- 대상 행사
- 대상 학교
- 수신자 이메일
- 제목
- 발송 예정일
- 예약 상태

주요 버튼:

- `내용 수정`
- `발송일 수정`
- `수신자 수정`
- `미리보기`
- `테스트 발송`
- `즉시 발송`
- `예약 취소`

### 13.5 발송 이력 탭

목적:

- 이미 발송된 메일을 확인한다.

표시:

- 메일 유형
- 대상 학교
- 수신 이메일
- 발송일
- 발송 상태
- 열람 여부
- 재발송 횟수

주요 버튼:

- `메일 보기`
- `재발송`
- `다른 이메일로 재발송`

### 13.6 수신자 목록 탭

목적:

- 메일 발송 대상자를 확인하고 수정한다.

표시:

- 학교명
- 담당교사명
- 기본 이메일
- 추가 수신 이메일
- 수신 제외 여부
- 최근 발송 상태

주요 버튼:

- `이메일 수정`
- `추가 수신자 등록`
- `수신 제외`
- `수신자 엑셀 다운로드`

수신자 정책:

- 기본 수신자는 선생님 회원 이메일이다.
- 선생님 회원가입 시 업무용 이메일 사용을 권장한다.
- 같은 내용의 메일은 여러 선생님에게 한 번에 보낼 수 있다.
- 추가 수신자를 등록할 수 있다.
- 특정 수신자를 발송 대상에서 제외할 수 있다.

### 13.7 발신자 설정 탭

목적:

- 메일 발신자 이름과 발신자 주소를 관리한다.

표시:

- 기본 발신자 이름
- 기본 발신자 이메일
- 행사별 발신자 이메일
- 최근 수정일
- 수정자

주요 버튼:

- `발신자 수정`
- `테스트 발송`
- `기본값으로 복원`

운영자에게 중요한 점:

- 발신자 주소는 관리자가 직접 관리할 수 있다.
- 단, 실제 메일 발송 도메인 인증은 시스템 설정에서 별도로 관리되어야 한다.

### 13.8 운영자에게 중요한 점

자동 메일도 발송 전에는 반드시 확인하고 수정할 수 있어야 한다.

발송 완료된 메일은 원본을 수정하지 않고 재발송으로 처리한다.

메일 문구 초안은 단계별 템플릿으로 저장하고, 관리자가 추가/삭제/수정할 수 있어야 한다.

## 14. 문의/요청

### 14.1 페이지 목적

선생님이 보낸 요청을 확인하고 처리하는 화면이다.

### 14.2 요청 유형

- 신청 상태 문의
- 학교명 변경 요청
- 출품 소속명/팀명 변경 요청
- 신청 작품 수 변경 요청
- 작품 확인 요청
- 활동확인서 수정 요청
- 활동확인서 재발급 요청
- 심사표 문의
- 간식비/지원 문의

### 14.3 화면 구성

요청 리스트:

- 요청일
- 학교명
- 행사명
- 요청 유형
- 내용 요약
- 상태
- 담당자

상세:

- 요청 내용
- 관련 신청/작품/문서 정보
- 이전 처리 이력
- 답변 입력

### 14.4 주요 버튼

`요청 확인`

- 관리자가 요청을 확인 상태로 바꾼다.

`승인`

- 요청을 승인한다.

`반려`

- 요청을 반려하고 사유를 남긴다.

`답변 등록`

- 선생님에게 보일 답변을 등록한다.

`처리 완료`

- 요청을 완료 처리한다.

### 14.5 운영자에게 중요한 점

전화나 이메일로 흩어지는 문의를 가능한 한 이 화면에서 구조화해 처리하는 것이 좋다.

그래야 누가 언제 어떤 요청을 처리했는지 이력이 남는다.

## 15. 연동 설정

### 15.1 페이지 목적

29초영화제/29역숏폼왕 외부 관리자 페이지와 29 WITH를 연결하는 화면이다.

### 15.2 화면 구성

표시:

- 행사명
- 외부 관리자 페이지 URL
- 외부 행사 식별값
- 서버 접근 정보 등록 여부
- 필드 매핑 상태
- 마지막 동기화 시간
- 연동 상태

필수 매핑:

- 행사 식별값
- 소속명/팀명
- 작품명
- 감독명/출품자명
- 출품 URL
- 출품일
- 평점
- 수상결과

### 15.3 주요 버튼

`연동 정보 입력`

- 외부 관리자 페이지 URL, 서버 정보, 행사 식별값을 입력한다.

`접속 테스트`

- 서버 접근이 가능한지 확인한다.

`필드 매핑 확인`

- 소속명/팀명, 작품명, 점수 등 필요한 데이터가 있는지 확인한다.

`동기화 실행`

- 외부 출품작 데이터를 가져온다.

`연동 로그 보기`

- 연동 성공/실패 이력을 확인한다.

### 15.4 팝업이 필요한 경우

다른 행사 데이터 감지:

```text
선택한 행사와 다른 출품 데이터가 감지되었습니다.
동기화를 중단하고 연동 설정을 확인해 주세요.
```

### 15.5 운영자에게 중요한 점

외부 관리자 페이지 URL만으로는 안정적인 자동 확인이 어렵다.

서버 접근 정보와 외부 행사 식별값, 필드 매핑이 함께 필요하다.

29초영화제와 29역숏폼왕 서버 접근 정보는 별도로 관리할 수 있어야 한다.

같은 서버 정보를 계속 사용하더라도 행사별로 연결 상태를 확인할 수 있어야 안전하다.

## 28. 페이지 간 연결 요약

### 28.1 신청 흐름

```text
신청 가능한 행사

→ 신청서 작성
→ 신청 완료
→ 선생님 대시보드 프로젝트 카드 생성
→ 관리자 신청/선정 리스트 표시
```

### 28.2 선정 흐름

```text
관리자 선정 처리
→ 선정 메일 발송
→ 구독권 지급
→ 담당자 페이지 생성
→ 작품 슬롯 생성
→ 선생님 대시보드 선정 완료 표시
```

### 28.3 출품 확인 흐름

```text
외부 출품작 동기화
→ 행사 ID 기준 데이터 수집
→ 출품 소속명/팀명 완전 일치 자동 매칭
→ 학교별 출품 확인
→ 미확인 건 선생님 확인 요청
```

### 28.4 활동확인서 흐름

```text
출품 마감
→ 관리자 최종 리스트 승인
→ 활동확인서 PDF 생성
→ 발급 메일 발송
→ 선생님 다운로드
```

### 28.5 심사표 흐름

```text
수상작 발표
→ 심사 결과 동기화
→ 순위/본심 여부 반영
→ 심사표 PDF 생성
→ 결과 안내 메일 발송
→ 선생님 다운로드
```

### 28.6 종료 흐름

```text
종료 전 점검
→ 미완료 항목 처리
```

## 29. 운영자가 꼭 기억해야 할 규칙

1. 여러 꿈프가 동시에 진행될 수 있다.
2. 항상 현재 선택한 꿈프 행사명을 확인해야 한다.
3. 학교명만으로 출품작을 매칭하면 안 된다.
4. 자동 매칭은 출품 소속명/팀명 완전 일치만 허용한다.
5. 선정 처리는 메일, 쿠폰, 담당자 페이지, 작품 슬롯을 함께 생성한다.
6. 자동 메일은 발송 전 내용, 날짜, 수신자를 확인할 수 있어야 한다.
7. 활동확인서는 관리자 최종 승인 후에만 발급된다.
8. 심사표는 수상작 발표 후 제공된다.
9. 29역숏폼왕에는 본심 여부를 표시하지 않는다.
10. 꿈프 종료 후에도 선생님은 문서를 다운로드할 수 있어야 한다.
