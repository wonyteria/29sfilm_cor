# 29 WITH - 꿈프 관리자 예외 행동 및 팝업/오류 안내 설계

## 1. 설계 목적

꿈프 관리자가 운영 중 실수하거나, 자동 처리 일부가 실패하거나, 예외적으로 수동 조치를 해야 하는 상황을 미리 정의한다.

목표:

- 영향이 큰 작업은 실행 전 반드시 확인한다.
- 자동 처리 일부 실패 시 전체 흐름이 끊기지 않게 한다.
- 어떤 항목이 성공했고 어떤 항목이 실패했는지 관리자가 바로 알 수 있게 한다.
- 관리자 수동 조치는 모두 이력으로 남긴다.
- 선생님 화면에 불필요한 내부 오류가 노출되지 않게 한다.

팝업 사용 기준:

- 상태가 바뀌는 작업
- 메일이 발송되는 작업
- 쿠폰/간식비/문서처럼 혜택이나 증빙에 영향을 주는 작업
- 수동 매칭처럼 잘못 연결될 위험이 있는 작업
- 종료처럼 되돌리기 어려운 작업

---

## 2. 관리자 예외 상황 분류

관리자 예외 상황은 아래 10개로 나눈다.

1. 행사 생성/수정 예외
2. 모집 시작 예외
3. 신청/선정 처리 예외
4. 메일/예약 발송 예외
5. 외부 연동/동기화 예외
6. 출품작 수동 매칭 예외
7. 활동확인서 예외
8. 심사 결과/심사표 예외
9. 혜택/쿠폰/간식비 예외
10. 종료/히스토리 예외

---

## 3. 행사 생성/수정 예외

### 3.1 필수 정보가 빠진 상태로 저장

상황:

- 행사명, 행사 유형, 공모기간, 출품 URL, 출품 마감일 등 필수값 누락

처리:

- 저장은 임시저장 가능
- 모집 시작은 불가

화면 안내:

```text
모집 시작에 필요한 정보가 부족합니다.
누락 항목을 입력한 후 꿈프 모집을 시작할 수 있습니다.
```

버튼:

- `누락 항목 보기`
- `임시저장`

팝업:

- 단순 저장에는 사용하지 않음
- 모집 시작 시 누락되었을 때 사용

---

### 3.2 이미 모집 중인 행사 정보를 수정

상황:

- 모집 시작 후 공모기간, 출품 마감일, 출품 URL 등을 수정하려 함

위험:

- 예약 메일 날짜 변경
- 선생님 안내 내용 변경
- 외부 연동 기준 변경

팝업 문구:

```text
모집 중인 행사 정보를 수정하시겠습니까?

수정 내용에 따라 예약 메일, 선생님 안내, 출품 확인 일정이 달라질 수 있습니다.
```

버튼:

- `수정 계속`
- `취소`

추가 확인:

- 출품 마감일 변경 시 D-day 예약 메일 재계산 여부 선택

```text
출품 마감일이 변경되었습니다.
D-day 안내 메일 일정을 새 마감일 기준으로 다시 계산하시겠습니까?
```

버튼:

- `다시 계산`
- `기존 일정 유지`

이력:

- 변경 전/후 값
- 수정자
- 수정일

---

### 3.3 행사 유형 변경

상황:

- 29초영화제에서 29역숏폼왕으로 변경 또는 반대

위험:

- 소속명/팀명 매칭 기준 변경
- 심사표 본심 여부 표시 기준 변경
- 외부 필드 매핑 변경

처리:

- 모집 시작 전에는 가능
- 모집 시작 후에는 권한 있는 관리자만 가능

팝업 문구:

```text
행사 유형을 변경하면 출품 확인 기준과 심사표 항목이 달라집니다.
정말 변경하시겠습니까?
```

버튼:

- `행사 유형 변경`
- `취소`

---

## 4. 모집 시작 예외

### 4.1 외부 연동 설정 없이 모집 시작

상황:

- 외부 URL 또는 서버 정보가 아직 없음

정책:

- 모집 시작은 가능
- 자동 출품 확인은 비활성

팝업 문구:

```text
외부 연동 정보가 아직 완료되지 않았습니다.

모집은 시작할 수 있지만, 출품작 자동 확인은 연동 설정 완료 후 사용할 수 있습니다.
```

버튼:

- `모집 시작`
- `연동 설정으로 이동`
- `취소`

---

### 4.2 모집 안내 메일 수신자 없음

상황:

- 꿈프 회원이 없거나 수신 가능한 이메일이 없음

팝업 문구:

```text
모집 안내 메일을 보낼 수신자가 없습니다.
신청 버튼은 활성화할 수 있지만 안내 메일은 발송되지 않습니다.
```

버튼:

- `신청 버튼만 활성화`
- `취소`

이력:

- 메일 미발송 사유 저장

---

### 4.3 모집 시작 실행 전 확인

상황:

- `꿈프 모집 시작` 클릭

팝업 문구:

```text
꿈프 모집을 시작하시겠습니까?

실행 후 아래 작업이 진행됩니다.
- 신청 버튼 활성화
- 행사 상태 모집중 변경
- 모집 안내 메일 예약 또는 발송
```

버튼:

- `모집 시작`
- `취소`

---

## 5. 신청/선정 처리 예외

### 5.1 쿠폰 부족 상태에서 선정

상황:

- 선정 대상 수보다 미사용 쿠폰 수가 적음

팝업 문구:

```text
구독권 쿠폰 수량이 부족합니다.

선정 처리는 가능하지만 일부 학교는 구독권이 지급 예정 상태로 표시됩니다.
```

표시 정보:

- 선정 대상 수
- 사용 가능 쿠폰 수
- 부족 수량

버튼:

- `쿠폰 없이 선정 계속`
- `쿠폰 업로드`
- `취소`

정책:

- 선정은 가능
- 쿠폰 미지급 학교는 관리자 작업함에 남김

---

### 5.2 중복 신청 가능성이 있는 학교 선정

상황:

- 같은 학교명, 같은 담당자 이메일, 비슷한 출품 소속명이 존재

팝업 문구:

```text
중복 신청 가능성이 있는 학교가 있습니다.
선정 전 신청 정보를 확인해 주세요.
```

버튼:

- `중복 후보 보기`
- `그래도 선정`
- `취소`

정책:

- 자동 병합하지 않는다.
- 관리자 확인 후 선정한다.

---

### 5.3 선정 처리 일부 실패

상황:

- 상태 변경 성공
- 메일 발송 실패
- 쿠폰 지급 실패
- 담당자 페이지 생성 실패

처리:

- 결과 팝업 표시

팝업 문구:

```text
선정 처리가 일부 완료되지 않았습니다.

완료:
- 신청 상태 선정 변경
- 담당자 페이지 생성

확인 필요:
- 선정 메일 발송 실패 2건
- 구독권 지급 실패 1건
```

버튼:

- `실패 건 처리`
- `닫기`

정책:

- 실패 항목은 관리자 작업함에 생성
- 선생님 화면에는 내부 실패를 직접 표시하지 않음

---

### 5.4 선정 상태 되돌리기

상황:

- 선정 처리 후 예비 또는 미선정으로 변경하려 함

위험:

- 이미 메일 발송됨
- 쿠폰 지급됨
- 담당자 페이지 생성됨

팝업 문구:

```text
선정 상태를 변경하시겠습니까?

이미 발송된 메일과 지급된 쿠폰, 생성된 담당자 페이지가 있습니다.
필요한 후속 조치를 확인해 주세요.
```

선택 항목:

- 안내 메일 재발송
- 구독권 회수 여부
- 담당자 페이지 비활성화 여부

버튼:

- `상태 변경`
- `취소`

권한:

- 최고 관리자 또는 꿈프 운영 관리자만 가능

---

## 6. 메일/예약 발송 예외

### 6.1 예약 메일 발송 전 필수 변수 누락

상황:

- 쿠폰번호 없음
- 활동확인서 링크 없음
- 심사표 링크 없음
- 출품 URL 없음

처리:

- 발송 버튼 비활성

화면 안내:

```text
메일 발송에 필요한 정보가 없습니다.
누락 정보를 확인한 후 발송할 수 있습니다.
```

버튼:

- `누락 정보 보기`

---

### 6.2 예약 메일 수신자 수정

상황:

- 수신 이메일 변경 또는 추가 수신자 등록

팝업 문구:

```text
수신자 정보를 수정하시겠습니까?
수정된 이메일로 예약 메일이 발송됩니다.
```

버튼:

- `수정 저장`
- `취소`

이력:

- 변경 전/후 이메일
- 수정자
- 수정일

---

### 6.3 즉시 발송

상황:

- 예약된 메일을 예정일 전에 바로 발송

팝업 문구:

```text
예약된 메일을 지금 발송하시겠습니까?
발송 후에는 내용을 수정할 수 없으며, 재발송만 가능합니다.
```

버튼:

- `즉시 발송`
- `취소`

---

### 6.4 예약 취소

상황:

- 예정된 메일 발송 취소

팝업 문구:

```text
예약 메일을 취소하시겠습니까?
취소된 메일은 자동 발송되지 않습니다.
```

버튼:

- `예약 취소`
- `닫기`

이력:

- 취소 사유 입력 권장

---

### 6.5 대량 메일 발송 전 확인

상황:

- 여러 학교에 메일 발송

팝업 문구:

```text
선택한 대상에게 메일을 발송하시겠습니까?
```

표시 정보:

- 대상 학교 수
- 수신자 수
- 제외 대상 수
- 메일 제목
- 발송 방식: 즉시/예약

버튼:

- `메일 발송`
- `미리보기`
- `취소`

---

## 7. 외부 연동/동기화 예외

### 7.1 서버 접근 정보 오류

상황:

- 로그인 실패
- 권한 없음
- 서버 응답 없음

관리자 화면:

```text
외부 서버에 접속하지 못했습니다.
서버 정보 또는 권한을 확인해 주세요.
```

버튼:

- `다시 테스트`
- `연동 정보 수정`
- `로그 보기`

선생님 화면:

- `관리자 확인 중`으로 표시

---

### 7.2 필수 필드 매핑 누락

상황:

- 29역숏폼왕 팀명 필드 없음
- 29초영화제 소속명 필드 없음
- 작품 URL 필드 없음

처리:

- 자동 동기화 비활성

화면 안내:

```text
출품작 자동 확인에 필요한 필드가 부족합니다.
필드 매핑을 완료한 후 동기화할 수 있습니다.
```

버튼:

- `필드 매핑 설정`

---

### 7.3 다른 행사 데이터가 감지됨

상황:

- 외부 행사 식별값과 가져온 데이터의 행사명이 불일치

팝업 문구:

```text
선택한 행사와 다른 출품 데이터가 감지되었습니다.
동기화를 중단하고 연동 설정을 확인해 주세요.
```

버튼:

- `연동 설정 확인`
- `동기화 중단`

정책:

- 이 경우 자동 동기화는 중단한다.

---

### 7.4 동기화 결과가 급격히 변함

상황:

- 이전 동기화 100편, 이번 동기화 10편
- 이전보다 데이터가 크게 감소

팝업 문구:

```text
이전 동기화 결과와 큰 차이가 있습니다.
외부 데이터 또는 연동 설정을 확인한 후 반영해 주세요.
```

버튼:

- `결과 검토`
- `반영 보류`

정책:

- 자동 반영하지 않고 검토 대기 상태로 둔다.

---

## 8. 출품작 수동 매칭 예외

### 8.1 유사 이름 후보 수동 매칭

상황:

- 방학중학교 / 방학 중학교

팝업 문구:

```text
선택한 출품작을 이 학교에 연결하시겠습니까?

자동 매칭 기준과 일치하지 않는 항목입니다.
잘못 연결하면 활동확인서와 심사표에 영향을 줄 수 있습니다.
```

표시 정보:

- 학교명
- 출품 소속명/팀명
- 외부 출품명
- 작품명
- 감독/출품자명

버튼:

- `수동 매칭`
- `취소`

이력:

- 매칭 전/후
- 관리자
- 사유

---

### 8.2 이미 다른 학교에 연결된 작품 매칭

상황:

- 같은 작품 URL 또는 외부 작품 ID가 다른 학교에 연결되어 있음

팝업 문구:

```text
이미 다른 학교에 연결된 작품입니다.
연결을 변경하시겠습니까?
```

표시 정보:

- 현재 연결 학교
- 변경 대상 학교

버튼:

- `연결 변경`
- `취소`

권장:

- 사유 입력 필수

---

### 8.3 수동 매칭 취소

상황:

- 잘못 연결한 출품작을 해제

팝업 문구:

```text
수동 매칭을 해제하시겠습니까?
해제하면 선생님 화면의 출품 확인 상태가 변경될 수 있습니다.
```

버튼:

- `매칭 해제`
- `취소`

## 9. 활동확인서 예외

## 14. 관리자 오류 안내 문구 원칙

관리자 오류 문구는 아래 구조를 따른다.

```text
무엇이 실패했는지
왜 확인이 필요한지
지금 할 수 있는 조치
```

좋은 예:

```text
선정 메일 2건이 발송되지 않았습니다.
수신자 이메일을 확인한 후 재발송할 수 있습니다.
```

피해야 할 예:

```text
오류가 발생했습니다.
```

## 16. 최종 판단

관리자의 예외 행동은 대부분 아래 네 가지에서 발생한다.

1. 자동 처리 버튼을 눌렀지만 일부만 성공함
2. 잘못된 대상을 선정, 매칭, 발송하려 함
3. 이미 완료된 상태를 수정하거나 되돌리려 함
4. 종료 전에 미완료 항목이 남아 있음

따라서 관리자 화면은 모든 중요한 작업에서 아래 구조를 제공해야 한다.

```text
실행 전 영향 확인
실행 결과 요약
실패 건 바로 처리
작업 이력 저장
선생님 화면 반영 여부 확인
```
