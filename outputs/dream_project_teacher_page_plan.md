# 29 WITH - 꿈프 담당 선생님 페이지 기획

## 1. 기획 방향

꿈프 담당 선생님 페이지는 운영자가 쓰는 관리 도구가 아니다.

선생님 입장에서 가장 중요한 질문은 아래와 같다.

- 지금 우리 학교가 잘 진행하고 있는가?
- 꿈프 신청이 정상적으로 접수되었는가?
- 놓친 안내나 해야 할 일이 있는가?
- 과거 참여 이력 때문에 우선순위 조정이나 패널티가 있는가?
- 성실 참여 이력으로 우선 검토나 혜택을 받을 수 있는 상태인가?
- 출품작이 정상 확인되었는가?
- 활동확인서, 심사표, 구독권, 간식비 등 필요한 혜택을 받을 수 있는가?
- 다음에 무엇을 해야 하는가?

따라서 선생님 페이지는 **확인과 안심 중심**으로 설계한다.

불필요한 정보:

- 관리자 내부 상태
- 외부 서버 연동 오류 상세
- 다른 학교 정보
- 전체 통계
- 관리자 메모
- 원본 점수
- 내부 예외 코드

선생님에게 필요한 정보:

- 내 신청 접수 상태
- 내 프로젝트 상태
- 내 참여 신뢰 상태
- 내 출품 확인 상태
- 내가 해야 할 일
- 받은 메일/공지
- 받을 혜택
- 다운로드 가능한 문서

---

## 2. 전체 메뉴 구조

선생님 페이지 메뉴:

```text
내 대시보드
참여 프로젝트
학생/작품 관리
공지/메일함
혜택/지원
활동확인서
심사표
참여 이력
내 프로필
```

메뉴 원칙:

- 선생님이 직접 조치할 수 있는 메뉴만 노출한다.
- 관리자 판단용 메뉴는 노출하지 않는다.
- 진행 중인 프로젝트가 없으면 신청 가능한 행사 안내를 보여준다.
- 과거 참여 상태는 `참여 이력`에서 확인하되, 중요한 안내는 대시보드에도 요약 표시한다.

---

## 3. 내 대시보드

첫 화면이다.

목적:

- 현재 참여 중인 꿈프 상태를 한눈에 확인
- 놓친 일이나 다음 할 일을 바로 확인
- 혜택 수령 상태를 빠르게 확인

표시 정보:

- 참여 중인 행사명
- 신청 접수 상태
- 선정 상태
- 참여 신뢰 상태
- 출품 마감일
- 마감 D-day
- 신청 작품 수
- 확인된 출품 작품 수
- 미확인 작품 수
- 다음 할 일
- 혜택 상태 요약

예시:

```text
제13회 박카스 29초영화제
신청 접수 완료
선정 완료
참여 상태: 성실 참여 학교
출품 마감 D-5

신청 5편 / 출품 확인 4편

다음 할 일
- 미확인 1편의 출품 상태를 확인해 주세요.
- 출품 시 소속명은 '방학중학교'로 입력되어야 합니다.
```

상태 카드:

- 신청 접수 완료
- 신청 검토 중
- 선정 완료
- 예비
- 미선정
- 성실 참여
- 확인 필요 이력 있음
- 정상 진행
- 확인 필요
- 마감 임박
- 출품 완료
- 활동확인서 발급 가능
- 심사표 발급 가능

UX 원칙:

- 숫자는 크게, 설명은 짧게 표시한다.
- 신청이 들어갔는지 여부는 첫 화면 상단에서 바로 확인 가능해야 한다.
- 과거 참여 이력은 불안감을 주는 표현보다 현재 상태와 필요한 조치를 중심으로 표시한다.
- 문제가 없으면 “정상 진행 중”을 명확히 보여준다.
- 문제가 있으면 가장 위에 조치가 필요한 항목만 보여준다.

### 3.1 신청 접수 상태

선생님은 신청 직후 “내 신청이 잘 들어갔는지”를 가장 먼저 확인하고 싶어 한다.

따라서 신청 상태를 대시보드 상단과 참여 프로젝트 목록에 모두 표시한다.

상태:

- 신청서 작성 전
- 신청 임시저장
- 신청 접수 완료
- 관리자 검토 중
- 선정 완료
- 예비
- 미선정
- 신청 취소

표시 예시:

```text
신청 접수 완료
2026.04.02 14:30에 신청서가 접수되었습니다.
관리자 검토 후 선정 결과가 안내됩니다.
```

```text
선정 완료
이 프로젝트의 담당자 페이지가 생성되었습니다.
출품 마감 전까지 학생 작품 출품 상태를 확인해 주세요.
```

신청 접수 후 선생님이 볼 수 있어야 하는 정보:

- 신청일
- 신청 행사명
- 예상 출품편수
- 출품 소속명/팀명
- 신청 상태
- 수정 가능 여부
- 선정 결과 안내 예정 상태

정책:

- 신청 완료 후에는 “신청 완료” 확인 화면과 확인 메일을 제공한다.
- 신청 상태가 바뀌면 공지/메일함에도 이력이 남아야 한다.
- 선생님이 같은 행사에 중복 신청하지 않도록 이미 신청한 행사는 “신청 완료”로 표시한다.

### 3.2 참여 신뢰 상태

과거 참여 이력은 선생님에게도 일부 안내해야 한다.

다만 관리자 내부 평가처럼 보이면 부담스럽기 때문에, “패널티 점수”가 아니라 현재 참여 상태와 안내 문구로 보여준다.

표시 상태:

- 첫 참여
- 정상 참여
- 성실 참여
- 우선 검토 대상
- 확인 필요 이력 있음
- 우선순위 조정 가능

표시 예시:

```text
참여 상태: 성실 참여
이전 프로젝트에서 출품 완료 이력이 확인되어 향후 선정 시 긍정적으로 검토될 수 있습니다.
```

```text
참여 상태: 확인 필요 이력 있음
이전 프로젝트에서 출품 미완료 이력이 있어 이번 프로젝트에서는 실제 출품 가능 여부를 꼭 확인해 주세요.
```

선생님에게 보여줄 참여 이력:

- 신청 횟수
- 선정 횟수
- 출품 완료 횟수
- 미이행 횟수
- 최근 참여 행사
- 현재 안내 문구

노출하지 않는 정보:

- 관리자 내부 메모
- 내부 점수
- 다른 학교와의 비교 순위
- 선정 심사 내부 기준

정책:

- 성실 참여 학교는 “우선 검토 가능” 또는 “성실 참여 이력 있음”으로 긍정적으로 표시한다.
- 미이행 이력이 있는 경우도 비난하는 표현을 쓰지 않고, 이번 프로젝트에서 필요한 조치를 안내한다.
- 관리자는 학교별 참여 이력을 수정하거나 보정할 수 있어야 한다.
- 선생님이 이력에 이의가 있으면 문의 또는 확인 요청을 보낼 수 있어야 한다.

---

## 4. 참여 프로젝트

선생님이 참여 중이거나 과거 참여했던 꿈프를 확인하는 화면이다.

구분:

- 진행 중
- 종료

진행 중 프로젝트 표시:

- 행사명
- 행사 유형
- 신청 접수 상태
- 선정 상태
- 출품 마감일
- 출품 확인 상태
- 다음 할 일

종료 프로젝트 표시:

- 행사명
- 출품 완료 여부
- 활동확인서 발급 여부
- 심사표 발급 여부
- 다운로드 바로가기

정책:

- 종료된 프로젝트도 선생님이 다시 확인하고 문서를 다운로드할 수 있어야 한다.
- 관리자가 꿈프 종료 처리한 경우 해당 프로젝트는 종료 영역으로 이동한다.

---

## 5. 학생/작품 관리

선생님이 학생들의 출품 상태를 확인하는 화면이다.

표시 방식:

- 작품 단위 리스트
- 신청 작품 수만큼 기본 슬롯 생성
- 자동 확인된 출품작은 자동으로 연결
- 미확인 작품은 직접 링크 입력 또는 상태 확인 안내

표시 항목:

- 번호
- 작품명
- 감독/출품자명
- 출품 URL
- 출품 확인 상태
- 문제 여부

상태:

- 확인 완료
- 확인 대기
- 직접 입력 필요
- 소속명/팀명 확인 필요
- 관리자 확인 중

선생님 조치:

- 작품 링크 직접 입력
- 작품명 확인
- 감독/출품자명 확인
- 소속명/팀명 안내 다시 보기

노출하지 않는 정보:

- 외부 관리자 페이지 원본 로그
- 자동 매칭 알고리즘
- 다른 학교 후보 작품

---

## 6. 공지/메일함

관리자가 발송한 주요 안내 메일을 다시 볼 수 있는 화면이다.

표시 메일:

- 모집 안내
- 선정 안내
- 출품 마감 D-14 안내
- 출품 마감 D-10 안내
- 출품 마감 D-5 안내
- 출품 마감 D-1 안내
- 출품 확인 완료 안내
- 활동확인서 발급 안내
- 심사 결과 반영 안내
- 간식비/지원 안내

표시 항목:

- 제목
- 발송일
- 읽음 여부
- 관련 프로젝트

정책:

- 이메일을 놓쳤더라도 페이지에서 다시 확인 가능해야 한다.
- 첨부나 링크가 있으면 페이지 안에서 다시 접근할 수 있어야 한다.
- 메일 발송 실패 여부는 선생님에게 노출하지 않는다.

---

## 7. 혜택/지원

선생님이 받을 수 있는 혜택과 지원 상태를 확인하는 화면이다.

표시 항목:

- 한국경제신문 구독권
- 간식비 또는 지원금
- 활동확인서
- 심사표

### 7.1 한국경제신문 구독권

표시:

- 지급 상태
- 쿠폰번호
- 지급일
- 사용 안내

상태:

- 지급 완료
- 지급 예정
- 지급 보류

### 7.2 간식비 또는 지원금

간식비 지급 예정일은 꿈프 관리자가 입력한다.

관리자 입력 항목:

- 지급 여부
- 지급 예정일
- 지급 금액
- 지급 기준
- 지급 상태
- 안내 메모

선생님 화면 표시:

- 지급 상태
- 지급 예정일
- 지급 금액
- 지급 안내

상태:

- 검토 중
- 지급 예정
- 지급 완료
- 보류
- 지급 대상 아님

예시:

```text
간식비
상태: 지급 예정
예정일: 2026.07.15
금액: 150,000원
안내: 출품 확인 완료 학교 대상으로 순차 지급됩니다.
```

UX 원칙:

- “언제쯤 받을 수 있는지”를 가장 먼저 보여준다.
- 보류 또는 대상 아님인 경우 이유를 짧게 표시한다.
- 내부 정산 정보는 노출하지 않는다.

---

## 8. 활동확인서

관리자가 출품작 리스트를 승인한 후 다운로드할 수 있다.

표시:

- 발급 상태
- 발급일
- 포함 작품 수
- PDF 다운로드 버튼

상태:

- 발급 전
- 관리자 확인 중
- 다운로드 가능
- 재발급 준비 중

문구 예시:

```text
관리자 최종 확인 후 활동확인서를 다운로드할 수 있습니다.
```

```text
활동확인서가 발급되었습니다.
PDF를 다운로드해 주세요.
```

---

## 9. 심사표

수상작 발표 후 심사 결과가 반영되면 다운로드할 수 있다.

표시:

- 반영 상태
- 작품별 순위
- 본심 진출 여부
- PDF 다운로드 버튼

29초영화제:

- 순위
- 본심 진출 여부

29역숏폼왕:

- 순위
- 본심 없음

노출하지 않는 정보:

- 원본 예심 평점
- 원본 평점
- 심사 내부 값

---

## 10. 참여 이력

선생님이 우리 학교의 과거 꿈프 참여 상태를 확인하는 화면이다.

목적:

- 과거 신청/선정/출품 완료 이력을 확인
- 현재 우선 검토 또는 우선순위 조정 가능 상태를 이해
- 미이행 이력이 있다면 이번 프로젝트에서 무엇을 조심해야 하는지 확인

표시 정보:

- 신청 횟수
- 선정 횟수
- 출품 완료 횟수
- 미이행 횟수
- 최근 참여 프로젝트
- 현재 참여 상태
- 안내 문구

현재 참여 상태:

- 첫 참여
- 정상 참여
- 성실 참여
- 우선 검토 대상
- 확인 필요 이력 있음
- 우선순위 조정 가능

표시 예시:

```text
현재 참여 상태: 성실 참여
지난 프로젝트에서 신청 작품 수 대비 출품 완료가 확인되었습니다.
향후 프로젝트 선정 시 긍정적으로 검토될 수 있습니다.
```

```text
현재 참여 상태: 확인 필요 이력 있음
이전 프로젝트에서 출품 미완료 이력이 있습니다.
이번 신청 전 실제 출품 가능 여부를 확인해 주세요.
```

선생님 조치:

- 이력 확인 요청
- 관리자에게 문의
- 학교명 또는 담당자 변경 요청

정책:

- “패널티”라는 단어는 되도록 화면에 직접 노출하지 않는다.
- 불이익 가능성이 있는 경우에는 “우선순위 조정 가능”으로 표현한다.
- 긍정 이력은 “성실 참여”, “우선 검토 대상”처럼 선생님이 동기부여를 느낄 수 있게 표현한다.
- 이력 산정 기준은 짧게 공개하되, 내부 메모나 점수는 노출하지 않는다.

## 11. 내 프로필

선생님의 기본 정보와 학교 정보를 관리한다.

표시:

- 이름
- 이메일
- 휴대폰번호
- 학교명
- 학교급
- 담당교사명
- 담당과목
- 연락처
- 통장사본
- 신분증

정책:

- 학교명은 출품 확인 기준이므로 임의 변경할 수 없다.
- 학교명 변경은 관리자 승인 후 가능하다.
- 통장사본/신분증은 업로드 상태만 표시하고 원본은 제한적으로 노출한다.

## 12. 선생님 화면의 핵심 UX

### 12.1 한 화면에서 답해야 하는 질문

내 대시보드는 아래 질문에 답해야 한다.

```text
꿈프 신청이 정상 접수되었는가?
선정 또는 검토 상태는 무엇인가?
우리 학교는 지금 잘 진행 중인가?
과거 참여 이력상 주의할 점이 있는가?
확인 안 된 작품이 있는가?
마감까지 며칠 남았는가?
내가 지금 해야 할 일이 있는가?
받을 혜택은 언제 받을 수 있는가?
다운로드 가능한 문서가 있는가?
```

### 12.2 정보 우선순위

1. 신청 접수/선정 상태
2. 문제/할 일
3. 참여 신뢰 상태
4. 출품 확인 상태
5. 마감일
6. 혜택/지원 상태
7. 다운로드 문서
8. 공지/메일

### 12.3 숨겨야 하는 것

- 관리자 작업함
- 외부 연동 실패 상세
- 다른 학교 정보
- 내부 메모
- 쿠폰 재고
- 메일 발송 실패 로그
- 원본 심사 점수

```text
상단

- 선정 상태
- 참여 신뢰 상태

- 신청 작품 수

- 미확인 수

- 확인 대기 작품
```

## 14. 관리자와 선생님 화면 연결

관리자 액션이 선생님 화면에 미치는 영향:

- 선생님이 신청 완료 → 대시보드에 신청 접수 완료 표시
- 관리자가 선정 처리 → 선생님 페이지에 선정 프로젝트 생성
- 관리자가 참여 이력 확인 → 대시보드와 참여 이력에 상태 표시
- 관리자가 쿠폰 지급 → 혜택/지원에 구독권 표시
- D-day 메일 발송 → 공지/메일함에 표시
- 출품 자동 매칭 → 학생/작품 관리에 출품 확인 표시
- 관리자 최종 승인 → 활동확인서 다운로드 가능
- 심사 결과 반영 → 심사표 다운로드 가능
- 간식비 지급 예정일 입력 → 혜택/지원에 예정일 표시
- 꿈프 종료 → 참여 프로젝트의 종료 영역으로 이동
