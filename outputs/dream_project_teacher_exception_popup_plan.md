# 29 WITH - 꿈프 선생님 예외 행동 및 팝업/오류 안내 설계

## 1. 설계 목적

선생님이 꿈프를 신청하고 운영하는 과정에서 발생할 수 있는 예외 행동, 입력 실수, 혼동 지점을 미리 정의한다.

목표:

- 선생님이 실수하기 전에 막는다.
- 실수했을 때 무엇을 해야 하는지 바로 안내한다.
- 관리자 확인이 필요한 상황은 선생님에게 내부 오류처럼 보이지 않게 처리한다.
- 불필요한 팝업은 줄이고, 중요한 순간에만 확실히 안내한다.

팝업 사용 기준:

- 제출, 변경, 취소처럼 되돌리기 어렵거나 영향이 큰 행동
- 출품 확인 실패처럼 선생님이 즉시 알아야 하는 문제
- 학교명/소속명/팀명처럼 자동 매칭에 큰 영향을 주는 입력
- 개인정보/증빙 업로드 오류
- 마감 임박 또는 마감 후 상태 변경

팝업을 쓰지 않는 경우:

- 단순 안내
- 완료된 상태 확인
- 반복적으로 보이는 일반 정책
- 화면 안에서 바로 해결 가능한 작은 입력 오류

---

## 2. 예외 상황 분류

선생님 예외 상황은 아래 9개로 나눈다.

1. 회원가입/로그인 예외
2. 프로필 작성 예외
3. 신청서 작성/제출 예외
4. 신청 후 상태 확인 예외
5. 선정 후 운영 예외
6. 출품 소속명/팀명 불일치 예외
7. 학생/작품 관리 예외
8. 혜택/지원 예외
9. 활동확인서/심사표/종료 후 예외

---

## 3. 회원가입/로그인 예외

### 3.1 회원 유형을 잘못 선택한 경우

상황:

- 선생님이 `서포터즈`로 가입했는데 꿈프 신청을 하려고 함

화면 처리:

- 신청하기 클릭 시 팝업 노출

팝업 문구:

```text
꿈나무양성프로젝트 회원만 신청할 수 있습니다.
회원 유형 변경이 필요하면 관리자에게 요청해 주세요.
```

버튼:

- `관리자에게 요청`
- `닫기`

관리자 연계:

- 문의/요청에 `회원 유형 변경 요청` 생성

---

### 3.2 이메일 인증 또는 로그인 정보 오류

상황:

- 이메일 형식 오류
- 이미 가입된 이메일
- 비밀번호 오류

화면 처리:

- 팝업보다 입력란 하단 오류 문구 사용

문구:

```text
이미 가입된 이메일입니다.
로그인하거나 비밀번호를 찾아주세요.
```

버튼:

- `로그인`
- `비밀번호 찾기`

---

## 4. 프로필 작성 예외

### 4.1 학교명을 입력하지 않음

상황:

- 프로필 저장 또는 신청서 제출 시 학교명이 없음

화면 처리:

- 입력란 하단 오류 문구
- 제출 버튼 비활성화

문구:

```text
학교명은 필수 입력 항목입니다.
```

---

### 4.2 학교명을 나중에 바꾸려고 함

상황:

- 신청 후 선생님이 학교명을 수정하려고 함

팝업 문구:

```text
학교명은 출품작 자동 확인 기준으로 사용됩니다.
신청 후 학교명 변경은 관리자 승인 후 가능합니다.
```

버튼:

- `변경 요청`
- `취소`

관리자 연계:

- 문의/요청에 `학교명 변경 요청` 생성

---

### 4.3 통장사본/신분증 업로드 실패

상황:

- 파일 형식 오류
- 용량 초과
- 업로드 실패

화면 처리:

- 파일 영역 안 오류 문구

문구:

```text
파일을 업로드하지 못했습니다.
PDF, JPG, PNG 형식으로 다시 업로드해 주세요.
```

버튼:

- `다시 업로드`

팝업 사용:

- 반복 실패 시에만 팝업

반복 실패 팝업:

```text
파일 업로드가 계속 실패하고 있습니다.
나중에 다시 시도하거나 관리자에게 문의해 주세요.
```

버튼:

- `관리자에게 문의`
- `닫기`

---

## 5. 신청서 작성/제출 예외

### 5.1 출품 소속명/팀명 미입력

상황:

- 출품 소속명/팀명 없이 신청서 제출

화면 처리:

- 제출 버튼 비활성화
- 입력란 하단 오류 문구

문구:

```text
출품 소속명/팀명은 필수 입력 항목입니다.
실제 출품 시 입력할 이름과 띄어쓰기까지 동일하게 입력해 주세요.
```

---

### 5.2 출품 소속명/팀명이 학교명과 다름

상황:

- 학교명: 방학중학교
- 출품 소속명: 방학 중학교

처리:

- 신청서 제출 전 확인 팝업
- 자동으로 막지는 않지만 강하게 확인

팝업 문구:

```text
학교명과 출품 소속명/팀명이 다릅니다.

출품작 자동 확인은 출품 소속명/팀명을 기준으로 진행됩니다.
실제 출품 시에도 아래 이름을 띄어쓰기까지 동일하게 입력해야 합니다.

출품 소속명/팀명: 방학 중학교
```

버튼:

- `수정하기`
- `이 이름으로 신청`

정책:

- 학교명과 다를 수 있는 경우도 있으므로 강제 차단하지 않는다.
- 다만 이 확인 팝업은 반드시 보여준다.

---

### 5.3 예상 출품편수가 0 또는 너무 큼

상황:

- 0편 입력
- 비정상적으로 큰 수 입력

처리:

- 0편은 제출 불가
- 큰 수는 확인 팝업

0편 문구:

```text
예상 출품편수는 1편 이상 입력해 주세요.
```

큰 수 팝업:

```text
예상 출품편수가 많습니다.
실제 출품 가능한 작품 수인지 다시 확인해 주세요.
```

버튼:

- `수정하기`
- `그대로 신청`

권장 기준:

- 30편 이상 입력 시 확인 팝업
- 기준값은 관리자 설정 가능

---

### 5.4 모집 목표 작품 수 초과

상황:

- 신청 작품 수 합계가 행사 목표 작품 수를 초과함

처리:

- 신청은 가능
- 제출 전 안내 팝업

팝업 문구:

```text
현재 모집 예정 작품 수를 초과했습니다.

신청은 가능하지만 운영 상황에 따라 최종 선정되지 않을 수 있습니다.
```

버튼:

- `신청 계속하기`
- `취소`

---

### 5.5 중복 신청 시도

상황:

- 같은 선생님 또는 같은 학교가 같은 행사에 이미 신청함

처리:

- 신청하기 버튼을 `신청 완료`로 변경
- 클릭 시 안내 팝업

팝업 문구:

```text
이미 신청한 행사입니다.
신청 내용은 내 대시보드에서 확인할 수 있습니다.
```

버튼:

- `내 대시보드로 이동`
- `닫기`

관리자 검토:

- 같은 학교 다른 담당자가 중복 신청할 수 있으므로 관리자 화면에 중복 가능성 표시

---

### 5.6 신청서 제출 직전 최종 확인

상황:

- 선생님이 신청서 제출 버튼 클릭

처리:

- 최종 확인 팝업

팝업 문구:

```text
신청서를 제출하시겠습니까?

제출 후 학교명과 출품 소속명/팀명 변경은 관리자 승인 후 가능합니다.
```

확인 항목:

- 행사명
- 학교명
- 출품 소속명/팀명
- 예상 출품편수

버튼:

- `신청서 제출`
- `다시 확인`

---

## 6. 신청 후 상태 확인 예외

### 6.1 신청 후 대시보드에 프로젝트가 안 보이는 경우

상황:

- 네트워크 지연 또는 생성 실패

처리:

- 신청 완료 화면에서 프로젝트 카드 생성 실패 시 안내

문구:

```text
신청은 접수되었지만 대시보드 반영이 지연되고 있습니다.
잠시 후 다시 확인해 주세요.
```

버튼:

- `새로고침`
- `관리자에게 문의`

관리자 연계:

- 신청 접수는 되어 있으나 담당자 페이지 생성 실패 로그 생성

---

### 6.2 미선정 상태 확인

상황:

- 선생님이 미선정 결과 확인

처리:

- 팝업보다 상태 화면 안내

문구:

```text
이번 프로젝트에는 선정되지 않았습니다.
다음 모집 시 다시 신청할 수 있습니다.
```

버튼:

- `다른 행사 보기`

팝업 사용:

- 미선정 상태로 변경된 최초 1회만 알림 팝업 가능

---

## 7. 선정 후 운영 예외

### 7.1 선정되었지만 구독권이 아직 지급되지 않음

상황:

- 쿠폰 재고 부족
- 관리자 지급 대기

선생님 표시:

```text
구독권 지급 준비 중입니다.
지급이 완료되면 이 화면과 메일로 안내됩니다.
```

팝업:

- 사용하지 않음

관리자 표시:

- 쿠폰 부족 또는 지급 실패를 작업함에 표시

---

### 7.2 선정 후 신청 작품 수를 변경하고 싶음

상황:

- 선생님이 5편 신청했지만 7편으로 늘리고 싶음

처리:

- 직접 수정 불가
- 요청 방식

팝업 문구:

```text
선정 후 신청 작품 수 변경은 관리자 확인이 필요합니다.
변경 요청을 보내시겠습니까?
```

버튼:

- `변경 요청`
- `취소`

관리자 연계:

- 문의/요청에 `신청 작품 수 변경 요청` 생성

---

## 8. 출품 소속명/팀명 불일치 예외

### 8.1 출품 확인 실패

상황:

- 자동 동기화 결과 해당 학교 출품작이 없음

선생님 화면 표시:

```text
아직 확인된 출품작이 없습니다.
출품 시 입력한 소속명/팀명이 아래 이름과 동일한지 확인해 주세요.

출품 소속명/팀명: 방학중학교
```

버튼:

- `소속명/팀명 복사`
- `작품 링크 입력`
- `관리자에게 확인 요청`

팝업:

- D-5 이하 또는 마감 후에도 미확인일 때 사용

D-5 팝업:

```text
출품 마감이 가까워졌지만 아직 확인되지 않은 작품이 있습니다.
학생들이 출품할 때 소속명/팀명을 정확히 입력했는지 확인해 주세요.
```

버튼:

- `작품 상태 보기`
- `나중에 확인`

---

### 8.2 선생님이 출품 소속명/팀명을 변경하려 함

상황:

- 신청 후 이름 변경 요청

팝업 문구:

```text
출품 소속명/팀명은 출품작 자동 확인 기준입니다.
변경하면 기존 출품작 확인이 지연될 수 있습니다.

관리자에게 변경 요청을 보내시겠습니까?
```

버튼:

- `변경 요청`
- `취소`

관리자 연계:

- 기존 매칭된 작품 영향도 표시

---

### 8.3 비슷한 이름의 출품작이 발견됨

상황:

- 완전 일치는 아니지만 공백 제거 시 유사한 이름이 있음
- 예: 방학중학교 / 방학 중학교

선생님 화면:

- 자동 매칭하지 않음
- “관리자 확인 중”으로 표시

관리자 화면:

- 유사 후보로 표시하고 수동 매칭 버튼 제공

선생님 문구:

```text
출품 정보 확인이 필요합니다.
관리자가 출품작 정보를 확인하고 있습니다.
```

팝업:

- 사용하지 않음

---

## 9. 학생/작품 관리 예외

### 9.1 작품 링크를 잘못 입력

상황:

- URL 형식이 아님
- 지원하지 않는 링크

처리:

- 입력란 하단 오류 문구

문구:

```text
올바른 작품 링크를 입력해 주세요.
```

지원 링크 예시:

- 29초영화제 출품 URL
- 29역숏폼왕 출품 URL
- YouTube
- Instagram
- TikTok
- Midibus

---

### 9.2 같은 작품 링크 중복 입력

상황:

- 작품 1과 작품 2에 같은 URL 입력

팝업 문구:

```text
이미 등록된 작품 링크입니다.
같은 작품을 중복 등록할 수 없습니다.
```

버튼:

- `다시 입력`

---

### 9.3 신청 작품 수보다 적게 출품

상황:

- 신청 5편, 확인 3편

화면 안내:

```text
신청한 작품 수보다 확인된 출품작이 적습니다.
마감 전까지 미확인 작품 상태를 확인해 주세요.
```

버튼:

- `작품 링크 입력`
- `관리자에게 확인 요청`

팝업:

- D-5, D-1, 마감 후에만 사용

---

### 9.4 신청 작품 수보다 많이 출품

상황:

- 신청 5편, 확인 7편

선생님 화면:

```text
신청 작품 수보다 많은 출품작이 확인되었습니다.
최종 활동확인서 포함 여부는 관리자 확인 후 결정됩니다.
```

버튼:

- `확인된 작품 보기`

관리자 화면:

- 초과 출품으로 표시
- 최종 포함 여부 선택

팝업:

- 사용하지 않음

### 9.5 학생 이름이 비어 있거나 이상함

상황:

- 감독명/출품자명이 누락됨
- 특수문자만 입력됨

선생님 화면:

```text
학생 이름 확인이 필요합니다.
활동확인서에 들어갈 이름이 맞는지 확인해 주세요.
```

버튼:

- `이름 확인 요청`
- `관리자에게 문의`

관리자 화면:

- 이름 확인 필요로 표시

## 13. 팝업 우선순위

팝업이 너무 많으면 사용성이 나빠진다.

따라서 아래 순서로 우선순위를 둔다.

### 13.1 반드시 팝업

- 신청서 최종 제출
- 학교명 변경 요청
- 출품 소속명/팀명 변경 요청
- 학교명과 출품 소속명/팀명이 다를 때 신청 제출
- 활동확인서 수정/재발급 요청
- 마감 후 작품 링크 제출

### 13.2 조건부 팝업

- D-5 이하 미확인 작품 있음
- D-0 마감 당일
- 간식비 지급 보류 최초 안내
- 예상 출품편수가 비정상적으로 큼
- 모집 목표 작품 수 초과

### 13.3 팝업 없이 화면 안내

- 신청 검토 중
- 구독권 지급 준비 중
- 간식비 지급 일정 확인 중
- 심사표 발표 전
- 활동확인서 승인 전
- 외부 연동 관리자 확인 중

### 13.4 토스트 메시지

- 쿠폰번호 복사
- 출품 소속명/팀명 복사
- 임시저장 완료
- 문의 접수 완료

---

## 14. 최종 보완 제안

기존 화면 설계에 아래 기능을 추가한다.

### 14.1 공통

- 팝업은 하루 1회 또는 상태 변경 최초 1회만 노출하는 옵션 필요
- 팝업 닫기 후 같은 안내는 대시보드에 작게 남김
- 모든 요청성 행동은 문의/요청 이력에 저장

### 14.2 선생님 화면

- 출품 소속명/팀명 복사 버튼
- 신청 완료 후 접수 정보 확인 화면
- 미확인 작품 D-day 경고
- 학교명/출품 소속명 변경 요청 버튼
- 활동확인서 수정 요청 버튼
- 마감 후 작품 링크 제출 안내

### 14.3 관리자 화면

- 학교명 변경 요청 처리
- 출품 소속명/팀명 변경 요청 처리
- 마감 후 제출 작품 검토
- 유사 이름 후보 수동 매칭
- 초과 출품 포함 여부 선택
- 팝업/공지 발송 이력 확인

---

## 15. 최종 판단

선생님의 예외 행동 대부분은 아래 네 가지에서 발생한다.

1. 신청이 제대로 되었는지 불안함
2. 학교명과 출품 소속명/팀명을 혼동함
3. 작품이 자동 확인되지 않아 당황함
4. 문서나 혜택이 언제 제공되는지 모름

따라서 선생님 화면은 모든 예외 상황에서 아래 구조를 유지해야 한다.

```text
현재 상태
왜 그런지 짧은 이유
지금 할 수 있는 행동 1개
필요하면 관리자에게 요청
```

팝업은 중요한 변경과 마감 임박 상황에만 사용하고, 나머지는 화면 안 안내로 처리하는 것이 가장 안정적이다.
